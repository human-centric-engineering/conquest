# F10.7 — Data-slot weighting, confidence, and a config preview

> Committable tracker for **F10.7**. Two workstreams on the report-research-and-data branch:
>
> 1. **Data-slot influence + confidence** — the AI report can now weigh the conversational **data-slot**
>    understanding against the **direct answers** (a soft 0–100 knob, default 50/50), include data-slot
>    **rationales**, and surface **confidence** so the writer can discount low-confidence items.
> 2. **Config preview** — a **Preview report** button renders how the configured report will read, from
>    AI-synthesised sample answers, before going live. Reuses the real generation core.

## Intent

- The narrative report drew **only** on the direct question answers; the parallel data-slot layer (with
  its own rationale + confidence) never reached the writer, and confidence was never surfaced.
- Admins had no way to see how a report configuration would read before publishing (no real responses
  exist pre-launch).

## What shipped

**Data-slot influence + confidence (both AI modes):**

- Two `generation` config knobs — `dataSlotInfluence` (0–100, default `DEFAULT_DATA_SLOT_INFLUENCE = 50`)
  and `discountLowConfidence` (default `true`) — wired type → default → narrow → Zod (optional, for
  import back-compat) → editor (slider + switch, gated/hinted on `dataSlotsEnabled`).
- `loadSessionExport` now loads `rationale` + `confidence` onto each `ExportDataSlotGroup` slot
  (additive; the respondent-facing "Captured information" appendix ignores them).
- New pure `buildDataSlotContextBlock` (`report/content.ts`) flattens filled data slots into a themed
  context block (paraphrase + `Why:` rationale + optional `(confidence x.xx)`); `buildAnswerTranscript`
  gained an `includeConfidence` option that annotates each `A:` line.
- `buildReportMessages` folds the data-slot block into the system prompt, adds the weighting instruction
  (`100 - dataSlotInfluence`% answers / `dataSlotInfluence`% data-slot context — soft), and, when
  `discountLowConfidence`, a discount-low-confidence instruction. Skipped entirely when the version has
  no data slots.

**Config preview (AI modes only):**

- `generateRespondentReport` split into a loader + the exported `generateReportFromInputs(inputs)` core.
- `synthesiseSampleReportInputs` (`report/preview-sample.ts`) invents one sample respondent via a
  structured LLM call and maps it through the **same** transcript + data-slot builders.
- New admin route `POST …/versions/:vid/report/preview` (`reportPreview` endpoint; `reportPreviewLimiter`,
  20/min/admin) — synthesise → force research + KB off → `generateReportFromInputs`. Rejects `raw` mode.
- Shared renderer extracted to `components/app/questionnaire/report/report-body.tsx`
  (`ReportBody` / `ReportPaperHeader` / `ReportDataAppendix` / …), re-imported by `session-complete.tsx`;
  the editor's **Preview report** button renders the result in a paper dialog behind a sample caveat.

## Decisions

- **Weighting is a soft prompt instruction**, not a deterministic filter — prose emphasis can't be
  enforced. It is inert without data slots (empty block ⇒ no instruction ⇒ pre-F10.7 behaviour).
- **Influence is independent of `rawIncludes`** — it feeds the _prompt_; `rawIncludes` controls the
  _displayed_ appendix. Feeding data slots as context is orthogonal to showing them.
- **Confidence is soft-disregard, not a hard floor** (per product decision) — surfaced + instructed,
  never a cutoff that silently drops answers.
- **Preview uses AI-synthesised answers** running the real generation core, so a previewed report and a
  live one share one path; web search + KB are forced off (fast/cheap/deterministic, no external deps).
  Only the AI modes preview (raw output = the answers, previewed via the respondent walkthrough).
- **No schema/migration** — both knobs live in the existing `respondentReport` JSON config column;
  data slots + fills load through existing relations.
- **Cohort-report preview deferred** — the extracted renderer, `generateReportFromInputs`, and the
  synthesiser are built to be reused for it later.

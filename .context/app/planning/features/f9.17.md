# F9.17 — Structure Edit Agent

> Committable tracker for **F9.17**. The Structure editor only let an admin change one entity at a
> time (rename a section, toggle a question's required, drag to reorder). There was no way to express a
> whole-document instruction — "renumber the sections", "CAPS every section title", "remove required
> from all free-text fields" — and have it applied across every matching question. This adds an "Edit
> with AI" panel that takes a plain-English instruction for the whole questionnaire, **previews the
> exact changes, and applies only on confirm**.

## Intent

Give the admin one instruction box that loops over the whole draft and makes the edits. The model's job
is interpretation, not data mutation: in the default **precise** mode it translates the instruction into
a list of deterministic edit-ops that execute in code, so mechanical edits never drift and untouched
fields are preserved. A second **rewrite** mode reuses the F9.8 refine capability for broad/semantic
changes. Both always preview-then-apply.

## What shipped

- **Edit-op core** (`lib/app/questionnaire/edit-agent/`): `edit-ops.ts` (Zod `EditOp` union +
  `Question`/`Section` selectors — `set_required`, `set_weight`, `transform_prompt`, `rename_prompt`,
  `transform_title`, `set_section_title`, `renumber_sections`, `reorder_sections`, `move_question`),
  `resolve.ts` (pure `resolveOps` — apply ops to a clone, diff against the original into a concrete
  `ResolvedChange[]`; throws `EditOpError` on an impossible op), `translate.ts` + `translate-prompt.ts`
  (one `runStructuredCompletion` turning the instruction → validated `EditPlan`).
- **New agent**: `app-questionnaire-structure-editor` (`AiAgent`, seed `060-edit-agent.ts`) — distinct
  budget + persona, empty model/provider (reasoning tier via `agent-resolver`). Registered in
  `AGENT_RECOMMENDATIONS`, so it appears on the **Agent Settings** page automatically (reasoning, 0.2,
  4096, medium effort) with the "Explain with AI" flow working.
- **Server pipeline** (`_lib/edit-agent-pipeline.ts`): `loadEditableStructure` (the precise-mode
  sibling of `loadRefinableStructure` — same draft + no-sessions guard, but returns ids + required +
  weight + ordinals) and `applyResolvedChanges` (granular per-entity updates in one transaction).
- **Routes**: `POST …/edit-agent/plan` (preview, no write — precise: translate→resolve; rewrite:
  dispatch refine capability, no persist) and `POST …/edit-agent/apply` (precise: re-resolve against
  live DB → granular updates; rewrite: `assertPersistable` → `replaceVersionStructure`). Admin-only,
  flag-gated, per-admin `composeLimiter`, `logAdminAction('questionnaire.edit_agent')`.
- **Flag**: `APP_QUESTIONNAIRES_EDIT_AGENT_ENABLED` (seed `059-edit-agent-flag.ts`, disabled by
  default) + `withEditAgentEnabled` + `resolveQuestionnaireWorkspaceFlags().editAgent`.
- **UI**: `edit-agent-panel.tsx` (instruction box, Precise/Full-rewrite toggle, Preview → before→after
  change list / new outline → Apply or Discard) wired into `version-editor.tsx`, shown only on a draft.

## Design notes

- **Two persistence paths, deliberately.** Precise mode does NOT route through
  `replaceVersionStructure` — that rewrites the whole graph and hardcodes `weight: 0.5` / forces
  `required → optional`, which would clobber tuned fields on every edit. It applies granular updates
  that touch only the named fields. Rewrite mode keeps `replaceVersionStructure` (acceptable for a full
  regenerate).
- **Preview is advisory.** Apply re-loads + re-resolves against live DB state, so a concurrent edit
  can't be silently clobbered; the client only round-trips the op list (precise) or the previewed
  structure (rewrite).
- **No new Prisma model / migration** — reuses the existing section/slot rows and the F9.8 refine
  capability.

## Not in scope

- Streaming SSE for the plan (shipped as a single structured completion — the F9.16 pattern).
- Per-item semantic prompt rewriting inside precise mode (use rewrite mode, or an explicit
  `rename_prompt`); adding/deleting sections or questions via instruction.

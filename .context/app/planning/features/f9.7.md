# F9.7 — Presentation mode (chat / form / both)

> Committable tracker for **F9.7**. A per-version config knob that lets an admin choose how a
> respondent completes a session: the streaming `chat`, a raw sectioned `form`, or `both` (a
> chat ↔ form toggle). The form doubles as an escape hatch when the chat struggles, and lets the
> respondent see and edit answers the agent inferred in the background.

## Intent

Until now a respondent could only complete a questionnaire through the conversation. Some
questionnaires (and some respondents) are better served by a raw form — sections of questions with
the right input per type — and the most powerful option is `both`: toggle between chat and form,
navigate sections, see a completeness map of what's already answered (incl. agent-inferred
answers), and edit any of it. Works in admin preview and real respondent modes.

## Decisions (confirmed with the user)

- **Default `chat`** — no behaviour change to existing launched versions; admins opt into form/both.
- **Manual edits are protected** — a respondent's form answer is authoritative; later chat-turn
  extraction/refinement must not silently overwrite it (contradiction detection may still warn).
- **Data slots + form = question-based, self-heal** — the form always edits the underlying
  questions (`AppAnswerSlot`); the chat's data-slot paraphrase reconciles on the next turn. Shipped
  with the brief divergence documented as a known limitation.
- **Edit-vs-fresh recording** reuses `refinementHistory` (new `'manual'` source): an entry =
  adjusted a pre-existing answer; no entry = answered fresh. Plus a `respondentEdited` boolean as
  the cheap pipeline-protection guard.
- **No new feature flag** — folds into the existing `APP_QUESTIONNAIRES_LIVE_SESSIONS_FLAG` gate
  (form mode adds no LLM spend).

## What shipped

- **Schema:** `AppQuestionnaireConfig.presentationMode` (`chat | form | both`, default `chat`) and
  `AppAnswerSlot.respondentEdited` (Boolean, default false). Migration
  `…_add_presentation_mode_and_respondent_edited` (create-only + phantom pgvector DDL stripped).
- **Config layer:** `PRESENTATION_MODES`/`PresentationMode` + `QuestionnaireConfigShape` field +
  default in `types.ts`; `updateConfigSchema` enum; `detail.ts` `CONFIG_SELECT`/`ConfigRow`/
  `toConfigView` narrowing; config-editor "Presentation mode" `<Select>`.
- **typeConfig client seam:** `lib/app/questionnaire/form/type-config.ts`
  (`readChoicesConfig`/`readLikertConfig`/`readNumericConfig`/`readBooleanConfig`) over the
  authoring schemas; `PanelSlotView.typeConfig` threaded through the pure builder + the DB read seam.
- **Form view read model:** `loadAnswerPanelState(…, forForm)` + `GET …/answers?view=form` — full
  structure regardless of `answerSlotPanelScope`, never the data-slot swap.
- **Write API:** `PUT …/answers` (`{ answers: [{ questionKey, value?, clear? }] }`) — flag →
  load → access (`resolveTurnAccess`) → active-status (409) → per-value `validateAnswerValue`
  (whole-batch reject) → one transaction → refreshed form view. Seam
  `_lib/form-answers.ts` (`recordManualAnswer`/`clearAnswer`, `'manual'` history source).
- **Protection:** `'manual'` added to `REFINEMENT_SOURCES`; `loadRespondentEditedSlotIds` + guards
  in `persistTurn` (`turn-run.ts`) skip overwriting respondent-edited slots.
- **Form UI** (`components/app/questionnaire/form/**`): `QuestionnaireForm`, `SectionNavigator`
  (completeness map, inferred markers), `QuestionField` dispatcher, app-local `RadioGroup` +
  `LikertScale`. Autosave hook `lib/hooks/use-form-answers.ts` (debounce + blur flush, empty→clear,
  `enabled` gate, `onSaved`).
- **Mode dispatch:** `session-workspace.tsx` three-way render + the `both` toggle (refresh form on
  enter, refetch panel on return); both server pages resolve `presentationMode` and seed the form
  view; `resolvePresentationModeForVersion` + `AnonymousSessionBoot` forward it to the no-login/
  preview surface.

## Not in scope

Eager data-slot paraphrase reconciliation on a form edit (self-heals next turn — follow-up); a
react-hook-form rewrite (controlled autosave is deliberate); per-flow PUT sub-cap (autosave is
debounced; inherits the 100/min section cap).

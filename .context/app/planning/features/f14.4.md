# F14.4 — Cohort Report deterministic scoring engine

> Committable tracker for **F14.4**. The "hard rules" layer: an admin defines a versioned scoring
> schema (scales · item→scale mappings with weight + reverse · band cutoffs) in a visual builder OR
> by extracting it from an uploaded document, and the deterministic engine turns answers into
> per-scale scores + bands that feed the cohort report's scored aggregation.

## Intent

Some questionnaires are scoring-led (Big Five and the like): answers map onto named scales, scales
get banded (Low/Mid/High), and the cohort is analysed by those scores. F14.4 adds that deterministic
"hard rules" engine alongside the AI narrative — fully reproducible, segmentable, and grounded in the
same k-anonymity floor as the rest of the report.

## Decisions

- **Both authoring paths, one schema model** — the visual builder and the upload-extract both
  produce/edit a single `AppScoringSchema.content` (`ScoringSchemaContent`). Upload returns a
  _proposal_ the admin reviews + saves through the builder's PATCH (no silent persist).
- **Versioned + forks on launch** — the schema is 1:1 with a version and copies on the launch fork
  (added to `copy-version-graph`), exactly like config/tags.
- **Pure engine, tested directly** — `scoreSession` (reverse on item bounds, weights, sum/mean,
  bands, normalisation) is pure; `compute.ts` is the only I/O (loads answers, scores, persists
  `AppRespondentScore`).
- **Scored aggregation reuses the segmentation groupings** — `buildCohortDataset` was refactored to
  build `DimensionGrouping`s once, shared by the per-question segmentation AND the per-scale segment
  means, under the same `K_ANONYMITY_THRESHOLD`.
- **Extraction reuses the cohort-report agent** — no new seeded agent; the extract pipeline supplies
  the version's real keys so every proposed item resolves, then prunes anything unmappable.
- **Recompute on save** — saving a schema recomputes `AppRespondentScore` for the version's sessions
  (best-effort; never fails the save).

## What shipped

- **Schema** — `AppScoringSchema` (versioned, fork-on-launch) + `AppRespondentScore` (per session,
  cascade); `scoringSchema`/`scores` back-relations; enums `SCORING_SCHEMA_SOURCES` /
  `SCORING_METHODS` / `SCORING_ITEM_SOURCES`. Migration `…_app_scoring_schema` (create-only, phantom
  pgvector DDL stripped, applied via `migrate deploy`).
- **Engine** — `scoring/types.ts` (schema + result types); `scoring/score.ts` (`scoreSession`);
  `scoring/schema-validation.ts` (`narrowScoringSchemaContent` + strict `scoringSchemaContentSchema`);
  `scoring/compute.ts` (`buildScoringInputs`, `scoreSessions`, `recomputeSessionScores`);
  `scoring/extract.ts` (`extractScoringSchema`); `scoring/index.ts` barrel.
- **Aggregation** — `buildCohortDataset` refactored to shared `DimensionGrouping`s; `buildScoring`
  adds the `CohortScoring` block (overall scale summaries + per-dimension segment means, k-anonymity
  suppressed); the report digest surfaces the scores.
- **API** — `GET`/`PATCH …/versions/:vid/scoring-schema` (read + builder save, fork-if-launched,
  recompute, audit); `POST …/scoring-schema/extract` (upload → proposal; sub-cap, audit). Endpoint
  helpers added.
- **UI** — `ScoringBuilder` (scales / item mappings / bands / method + Extract-from-document) on the
  new **Scoring** workspace tab (`flags.cohortReport`, page gated; nav + flag registry wired).

# F13.1 — Round Additional Context (interviewer briefing)

> Committable tracker for **F13.1**, the first of two cohort-round capabilities. An admin briefs the
> interviewer with facts/figures/background for a round — optionally per question — and the live
> interviewer quietly draws on it when asking. Off by default at every level. Built across phases
> 1–3 (shared foundation, backend, admin UI + AI authoring); Learning Mode (F13.2) shares phase 1.

## Intent

A questionnaire's questions are often about a subject the interviewer knows nothing about. Letting an
admin attach "briefing notes" — like prepping a human interviewer before they walk into the room —
lets the agent ask sharper, better-informed questions and follow up well, without changing the
questions themselves.

## Decisions

- **Round-level storage, per-question attribution** — entries live on the round
  (`AppRoundContextEntry`), scoped to a bundled `versionId`, optionally attributed to one
  `questionSlotId` (null = general). Cohort-specific facts live naturally per round.
- **Direct FK injection, no vector DB** — the interviewer already knows the selected question, so the
  briefing is a deterministic lookup (attributed entries for that question + general entries),
  injected verbatim. Auditable, zero per-turn embedding cost. (Vector retrieval was considered and
  rejected for v1 — see the doc.)
- **Two gates** — the platform `APP_QUESTIONNAIRES_ROUND_CONTEXT_ENABLED` flag (master AND cohorts
  AND this sub-flag) AND the per-round `contextEnabled` toggle. Nothing injects until both are on.
- **Briefing is the interviewer's, not the respondent's** — the prompt frames it as for-you-only:
  never read aloud, quoted, or attributed to the respondent.
- **AI/upload produce proposals, don't persist** — suggest + parse return data the admin reviews and
  saves through the normal create path (mirrors F12.2's discipline).

## What shipped

- **Schema** (phase 1) — `AppRoundContextEntry` (+ `AppRoundLearningDigest`, used by F13.2) and three
  `AppQuestionnaireRound` columns (`contextEnabled`, `learningEnabled`, `learningConfig`). Migration
  `…_app_round_context_and_learning` (create-only, phantom pgvector DDL stripped).
- **Flags** (phase 1) — `isRoundContextEnabled` / `ensure` / `with` gates; seed `050`. (F13.2's
  learning flag + seed `051` rode along in the same foundation commit.)
- **Backend** (phase 2) — CRUD routes under `…/rounds/:id/context` (+ `/:entryId`), the
  `buildRoundBriefing` read + pure `selectBriefingLines` selector, and a `<briefing>` section threaded
  into both phrasing branches of the live messages route (question + data-slot; data-slot resolves
  `mappedQuestionKeys` → ids).
- **Admin UI + AI** (phase 3) — `RoundContextPanel` (toggle, grouped list with provenance badges +
  attribution, add/edit with an attribution picker, delete, upload-to-text, AI suggest);
  `AppSuggestRoundBriefingCapability` (composer-agent-bound, seed `052`) + `…/context/suggest` and
  `…/context/parse` routes.

# F7.6 — Chat-transcript download

> Committable tracker for **F7.6**. The respondent can take their **conversation** away as a
> branded PDF or plain text — distinct from the F7.4 _answers_ export. An intro explains the
> questionnaire context + the support reference, and every turn is labelled
> ("Interviewer" / the respondent) and timestamped. No migration (reads existing turn rows).

## Intent

F7.4 lets the respondent download their captured **answers**. F7.6 adds the other half they
might want — the **verbatim conversation**: a record of what was asked and what they said, with
an intro that explains the questionnaire context and the chat reference, each turn timestamped
and attributed. Useful as a personal record, for accessibility, and for a respondent who wants
to keep proof of what they reported.

## Decisions (confirmed with the user)

- **Two formats, respondent's choice.** A "Transcript" dropdown offers **Themed PDF** (branded,
  same accent + logo treatment as the F7.4 answers PDF) and **Plain text** (`.txt`).
- **Intro / context block.** Both formats open with the questionnaire title, the support
  **reference** (`7F3K-9M2P`), version, goal, audience, the respondent line, start/completion
  timing, and status — then a short explainer of the labels and the UTC timestamp convention.
- **Speaker labels.** The agent is always **"Interviewer"**. The respondent is **their display
  name** when one is known AND the session is **not anonymous**; otherwise the generic
  **"Respondent"**. Anonymous mode never leaks identity into the label — the same redaction the
  F7.4 export applies, enforced in the pure builder.
- **Timestamps in UTC.** The export is generated server-side with no knowledge of the
  respondent's timezone; a fixed zone also keeps the pure builders deterministic for tests. The
  intro notes the UTC convention.
- **Two surfaces.** A quiet control on the **lifecycle bar** (available throughout the
  conversation once a real exchange exists, beside the ref chip) and on the **completion
  screen** (always, once submitted — independent of the responses-report config).

## What shipped

### Pure core (`lib/app/questionnaire/export/`, Prisma-free, Next-free)

- `transcript-types.ts` — `TranscriptExportModel` (intro metadata + resolved theme +
  labelled/timestamped `TranscriptTurnView[]`).
- `build-transcript-export-model.ts` — `buildTranscriptExportModel`: resolves the speaker
  labels (anonymous redaction), flattens turn rows (kickoff / empty lines skipped, matching the
  F7.1 replay), formats the reference, summarises the audience (shared `summariseAudience`), and
  resolves the theme. Clock-free (generation time passed in).
- `build-transcript-text.ts` — `buildTranscriptText`: the `.txt` serialiser (intro + labelled,
  timestamped turns; empty-conversation note).
- `transcript-format.ts` — shared deterministic UTC date/status formatting for both renderers.

### Document + render (server-only)

- `components/app/questionnaire/export/transcript-pdf-document.tsx` — the `@react-pdf/renderer`
  `<Document>` (branded intro header + per-turn label/timestamp/body, accent on interviewer
  turns, fixed footer with generation date + page numbers). Sibling to the F7.4
  `SessionPdfDocument`.
- `_lib/render-transcript-pdf.tsx` — `renderTranscriptPdf(model)` → `renderToBuffer(...)`.

### DB seam (`questionnaire-sessions/_lib/`)

- `transcript-export.ts` — `loadTranscriptExport(sessionId)` (one query: turns with timestamps,
  `publicRef`, version goal/audience, demo-client theme, `anonymousMode`, timing/status;
  respondent name looked up only when not anonymous) + `assembleTranscriptExportModel(loaded,
{ fetchLogo })` (PDF fetches the brand logo best-effort, text skips it; stamps `generatedAt`,
  delegates to the pure builder).
- `fetch-logo-data-uri.ts` — the best-effort logo→data-URI fetch, **extracted** from
  `session-export.ts` (F7.4) so both export seams share it (3 s timeout, 1 MB cap, https-only).
- `transcript-response.ts` — `transcriptPdfResponse` / `transcriptTextResponse`: `attachment`
  with `transcript-<slug>-v<N>.<ext>`, `Cache-Control: no-store`.

### Routes (dot-segment)

- `GET …/questionnaire-sessions/:id/transcript.pdf` (`runtime = 'nodejs'`) and
  `…/transcript.txt` — both `withLiveSessionsEnabled` + `resolveTurnAccess` (authed owner OR
  anonymous `X-Session-Token`). Gate order flag→load→access→build→render. Registry:
  `transcriptPdf` / `transcriptText` on `QUESTIONNAIRE_SESSIONS`.

### Respondent UI

- `lifecycle/transcript-download.tsx` — a `DropdownMenu` ("Themed PDF" / "Plain text"). Each
  option `fetch`es with the `X-Session-Token` header (no-login respondents have no cookie), saves
  the blob honouring the server `Content-Disposition` filename, and shows a transient inline
  error on failure. `ghost` variant on the lifecycle bar, `outline` on completion.
- `SessionLifecycleBar` gains a `download` slot (right cluster, beside the ref chip);
  `SessionWorkspace` mounts the control once `turnCount > 1` and chat is in play.
- `SessionComplete` shows the control beside the F7.4 responses download (always, once
  submitted).

- **Unit (render):** `render-transcript-pdf.test.tsx` — real `renderToBuffer` yields `%PDF` for

- **Integration (routes):** `transcript-export-routes.test.ts` — both routes' gate order, both
  access modes, response envelopes + filenames, `fetchLogo` true (PDF) / false (text), and the
  real text serialisation over the txt route.
- **Component:** `transcript-download.test.tsx` — each option hits the right route, token header
  only when anonymous, server-named blob download, inline error without download.

# F7.2 — Answer-slot panel

> Committable tracker for **F7.2**. The live panel beside the F7.1 chat showing the
> questionnaire's answer slots as the conversation fills them in: confidence indicators,
> refinement-history disclosure, and a per-slot "Revisit". One new read endpoint
> (`GET …/questionnaire-sessions/:id/answers`) plus consumer UI. Gated by
> `APP_QUESTIONNAIRES_ENABLED` + `APP_QUESTIONNAIRES_LIVE_SESSIONS_ENABLED`.

## Intent

F7.1 rendered the streamed conversation. F7.2 makes the agent's work **visible**: a
prospect watches answers land beside the chat, each with a quiet confidence dot and
provenance, expandable to its rationale and revision trail — and can ask to revisit any
answer. The panel is the demo's "look what it captured" beat.

## Decisions (confirmed with the user)

- **Click behaviour = inline-expand + Revisit.** Clicking an answered slot expands it
  (value, rationale, refinement history). A separate confirm-gated **Revisit** button
  sends an "I'd like to revisit…" message to the agent, re-asking through the same turn
  loop. We do **not** scroll the chat transcript to a turn — that would force touching
  `QuestionnaireChat` (which F7.1 kept untouched) and has a turn-ordinal alignment
  hazard.
- **Panel scope is an admin config setting.** New per-version `answerSlotPanelScope`
  (`full_progress` | `answered_only`, default `full_progress`). `answered_only` filters
  in the pure builder, so pending prompts never reach the client.
- **Rationale shown, quietly** — in the expanded view only.
- **Read endpoint, not a stream event.** The F6.1 SSE stream emits no slot events
  (answers persist server-side before `done`), so the panel reads
  `GET …/answers` and refetches when each turn settles. The streaming contract is
  untouched. See [decision: data seam](#data-seam).

## Data seam

`GET /api/v1/app/questionnaire-sessions/:id/answers` → `{ data: AnswerPanelView }`.
Gate order: **flag (404 pre-auth) → load → `resolveTurnAccess` (401/403) → respond**.
Reuses `resolveTurnAccess` for both respondent kinds (authed owner / anonymous
`X-Session-Token`). No status gate (paused/completed sessions still show answers); no
extra limiter (read). The pure `buildAnswerPanelView` does the join, the scope filter,
and count derivation; the `_lib` seam does one Prisma read (incl. a turn-id → ordinal
map for `answeredAtTurnIndex`). Authoring internals (`weight`, tags, config) are not
projected.

## Live wiring

`SessionWorkspace` (new client parent) owns the single `useQuestionnaireSessionStream`
and the `useAnswerPanel` fetch, and renders chat + panel side by side. The stream hook
gained an additive `onTurnSettled` (fired on a clean settle to `idle`) → `panel.refetch`.
`QuestionnaireChat` was refactored to **receive the stream via a `stream` prop** (hook
moved up), so chat and panel share one stream — which is what lets the panel's Revisit
call `stream.sendMessage(...)`. Render of the chat is otherwise unchanged.

## Layout

Both pages render `SessionWorkspace` inside the existing `BrandThemeProvider` (panel
inherits brand vars). Grid `lg:grid-cols-[1fr_22rem] xl:grid-cols-[1fr_26rem]`; panel is
`hidden lg:flex` so `< lg` keeps the full-width F7.1 chat. Authed page SSR-seeds the
panel (`loadAnswerPanelState`); the anonymous page can't (client-only token) so it
shows a brief skeleton.

## What shipped

**Config field (7-step path):** `answerSlotPanelScope` on `AppQuestionnaireConfig`
(schema + migration `…_add_config_answer_panel_scope`, generated `--create-only` with
the phantom pgvector DDL hand-stripped), `ANSWER_SLOT_PANEL_SCOPES` tuple + type +
default in `types.ts`, the `updateConfigSchema` enum, `CONFIG_SELECT`/`toConfigView`
narrowing, the config-editor `<Select>` + `<FieldHelp>`. The F2.1 fork-copy carries it
verbatim (CONFIG_SELECT spread).

**Read surface:** `lib/app/questionnaire/panel/{types,answer-panel,confidence}.ts`
(pure), `_lib/answer-panel.ts` (`loadAnswerPanelState`), the `answers` route, the
`endpoints.ts` registry entry.

**UI:** `use-answer-panel` hook; `onTurnSettled` on the stream hook; `QuestionnaireChat`
stream-as-prop refactor; `SessionWorkspace`; `panel/` components (`answer-slot-panel`,
`answer-slot-item`, `confidence-indicator`, `provenance-badge`, `refinement-history`,
`format-answer-value`); two-column layout on both pages + the anon boot swap.

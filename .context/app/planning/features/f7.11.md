# F7.11 — Respondent session resume

> Committable tracker for **F7.11**. Lets a respondent return to a session they already started —
> including a no-login respondent who closed the browser — instead of silently starting over.
> Gated by `APP_QUESTIONNAIRES_ENABLED` + the live-sessions flag (no dedicated resume flag; purely
> per-version config `sessionResumeEnabled`, default **on**).

## Intent

The no-login surface (`/q/[versionId]`) held its session credential only in `sessionStorage`
(per-tab, lost on browser close), and `createAnonymousSession` never resumes server-side — so
returning to the link **minted a fresh session and lost progress**. Authenticated respondents
resumed idempotently but had **no way to start fresh**. This feature makes returning a first-class,
chooseable action across three paths, plus a shared abandon.

## Decisions (confirmed with the user)

- **Reach: same-device AND cross-device by ref.** The device remembers its session (localStorage);
  a respondent can also resume from another device by entering their support code.
- **Who: anonymous AND authenticated.** The no-login path is the real gap; the authed path gains a
  Continue/Start-new chooser over its existing silent resume.
- **Gating: a Settings toggle, default on.** `sessionResumeEnabled` governs the whole capability; a
  kiosk/shared-device questionnaire can turn it off.
- **Same-tab refresh stays silent.** A per-tab marker distinguishes a refresh (silent resume) from a
  genuine return (the welcome-back chooser) — no nagging on reload.
- **by-ref is anonymous + resumable only, generic-404, hard-rate-limited.** The 8-char `publicRef`
  is a low-entropy bearer credential, so the guards keep the blast radius to "resume my own
  in-progress anonymous chat" and the endpoint is never an enumeration oracle.

## Build shape

- **Config** — `sessionResumeEnabled Boolean @default(true)` on `AppQuestionnaireConfig` (migration
  `20260715224104_add_session_resume_enabled`, hand-authored ADD COLUMN, phantom pgvector DROPs
  stripped; 5 vector indexes verified intact). Threaded through `types.ts`
  (`QuestionnaireConfigShape` + `DEFAULT_QUESTIONNAIRE_CONFIG`), `config-schema.ts`, `_lib/detail.ts`
  (`CONFIG_SELECT` / `ConfigRow` / `toConfigView`), `resolveSessionResumeEnabledForVersion`
  (`chat/anonymity.ts`), and a Settings-tab toggle in `config-editor.tsx`. Export/import/copy pick it
  up automatically (`CONFIG_KEYS` derived from the default; copy via `CONFIG_SELECT`).
- **Anonymous same-device** — `anon-session-storage.ts` (shared durable/ephemeral creds + per-tab
  marker helpers); `anonymous-session-boot.tsx` rewritten with the durable branch + `fetchStatus` +
  welcome-back state; `session-resume-gate.tsx` (Continue / Start new + progress + ref);
  `(public)/q/[versionId]/page.tsx` resolves + passes `resumeEnabled`.
- **Cross-device by ref** — `resume-by-ref/route.ts` (public, rate-limited) + `_lib/resume-by-ref.ts`
  (`resolveAnonymousResumeByRef` guard matrix) + `resumeByRefLimiter` (5/min) + `RESUME_BY_REF`
  endpoint constant; `resume-by-ref-form.tsx` + `resume-by-ref-entry.tsx` (welcome-back link + page
  footer).
- **Authenticated** — `findAuthedResumeDetail` (`chat/resumable-session.ts`); `start/page.tsx`
  renders `AuthedResumeChooser` for the versionId path when a session with real progress exists;
  `start/actions.ts` `startFreshAuthedSession` (abandon old + create fresh + redirect).
- **Shared abandon** — `[id]/lifecycle/route.ts` accepts `abandon` (authed owner + anonymous token;
  pause/resume stay signed-in only) driving `abandonSession` (already in `_lib/sessions.ts`).

## Scope notes

- Invite (`?i=`) and admin preview stay on the ephemeral `sessionStorage` path — invites already
  resume server-side via `invitationId`; preview is a single-per-version admin walkthrough.
- The authed chooser covers the **versionId** start path only; the invitation path keeps its
  idempotent silent resume (its round/cohort context is resolved by the create seam, not re-derived).
- Same-device resume relies on the HMAC token's 24h TTL. Returning after expiry (or on a new device)
  is the by-ref path.

## Verification

- Anonymous: answer 1–2 on `/q/<v>`, note the ref, close the tab, reopen → welcome-back; Continue
  replays, Start new abandons + fresh.
- Cross-device: enter the ref in another browser → resumes; bad/terminal/authed ref → generic 404;
  hammer → rate-limited.
- Authed: `/questionnaires/start?versionId=…` with an in-progress session → chooser.
- Toggle off on the Settings tab → anonymous returns mint fresh, by-ref 404s, no chooser.

# F14.9 — Definition import/export + blank-instrument download

> Committable tracker for **F14.9**. Two additive admin capabilities on a questionnaire version:
> (1) export the whole **definition** (structure + settings + data slots + scoring) as a portable
> JSON file and import it back as a brand-new questionnaire, and (2) download the **blank instrument**
> (the questions, for human review/printing) as PDF, plain text, or CSV.

## Intent

Admins could already move a version's run-time **settings** between versions (the Settings-tab
`config-import-export`), export **respondent results** (CSV/JSON), and download transcripts/reports
(PDF/text). Two gaps remained:

- **No way to move a questionnaire definition.** Cloning a questionnaire across environments, or
  keeping a versioned backup of the authored instrument, had no artifact. The definition (sections,
  questions, tags, config, data slots, scoring) lived only in the DB.
- **No printable blank form.** There was no human-readable rendering of the _empty_ instrument for
  review or paper distribution — only respondent-answer exports.

## Decisions

- **One full-fidelity envelope, mirroring `config-export`.** `definition-export.ts` is the
  structural sibling of the settings envelope: a `kind: 'conquest.questionnaire.definition'`
  discriminator + `schemaVersion`, `buildDefinitionExport` (reuses `extractConfig` so the config
  block never drifts), and `parseDefinitionImport` (the external-data boundary — `definitionImportSchema`
  Zod-validates an uploaded file before any of it reaches the DB). Scope = structure + config +
  data slots + scoring; **no respondent data, no embedding vectors**.
- **Import is always create-only.** A definition import creates a **brand-new** draft questionnaire
  (v1) owned by the importer — it never touches existing work. Implemented by a dedicated
  transactional persister (`_lib/import-definition.ts`), _not_ the upload-path `writeGraph` (which is
  lossy: hard-coded weight, no tags, `createMany` can't attach per-question tags).
- **Embeddings regenerated, never carried.** Question + data-slot pgvector embeddings are
  reproducible from text, so the envelope omits them; the import route regenerates them post-commit
  (best-effort `embedVersionSlots` / `embedVersionDataSlots`, with the runtime lazy self-heal as the
  backstop).
- **Cross-references by stable key.** Tags are remapped by `normalizedLabel`; data-slot↔question and
  scoring item/band refs resolve by question `key` (preserved on import), so nothing needs id
  rewriting.
- **Blank-instrument download = one model, three serialisers.** `build-instrument-model.ts` flattens
  the version graph into a presentation model consumed by text, CSV (formula-injection-safe via
  `csvEscape`), and React-PDF serialisers — sibling to the respondent-facing session/transcript
  exports but design-time and brand-free.
- **No new flag / schema / migration.** Reuses `withQuestionnairesEnabled`, the `exportLimiter`
  (downloads) and `ingestLimiter` (import) sub-caps, and existing models. App-scoped → **no CHANGELOG
  entry**.

## Surface

- **Lib:** `lib/app/questionnaire/authoring/definition-export.ts` (envelope build/parse/schema, barrel
  re-exported); `lib/app/questionnaire/export/build-instrument-{model,text,csv}.ts`.
- **PDF:** `components/app/questionnaire/export/instrument-pdf-document.tsx` +
  route-local `…/instrument/render-instrument-pdf.tsx`.
- **Persister:** `app/api/v1/app/questionnaires/_lib/import-definition.ts` (`persistDefinitionImport`).
- **Routes:** `…/versions/[vid]/definition` (GET JSON export); `…/versions/[vid]/instrument`
  (GET `?format=pdf|text|csv`, node runtime); `…/questionnaires/import` (POST import-as-new).
  Endpoint registry: `definitionImport`, `versionDefinition`, `versionInstrument`.
- **UI:** `definition-export-menu.tsx` (Structure-tab "Export / download" dropdown);
  `import-definition-dialog.tsx` (wired into `new-questionnaire-menu` as "Import definition").

## Verification

Type-check clean (full project; one pre-existing unrelated test-mock error). 20 new unit tests green
(envelope round-trip + reject paths, instrument model numbering/options/constraints, text + CSV
serialisers incl. formula-injection neutralisation). Manual round-trip: export questionnaire A →
import → new draft B with identical structure/tags/config/data-slots/scoring, A untouched, B
embeddings regenerated; PDF/text/CSV render the blank instrument with no respondent data.

# F10.5 ‚Äî Report web-search rounds

> Committable tracker for **F10.5**. Optional web-search rounds that bring **live external context** into
> report generation ‚Äî configured per questionnaire version, running **before** generation (to inform the
> report), **after** it (to enrich / fact-check), or **both**. Shipped for the Respondent Report; the
> module is report-kind-agnostic so the Cohort Report is an additive follow-up.

## Intent

- Reports were grounded only in the respondent's answers and the per-client knowledge base ‚Äî no way to
  pull in current, external facts (market data, regulations, recent events) at generation time.
- An admin should be able to describe, in plain English, what to search for and what to do with the
  results, per report version, without touching code.

## What shipped

1. **`web_search` capability** (`lib/app/questionnaire/capabilities/web-search.ts`) ‚Äî a provider-agnostic
   `BaseCapability`, query-in / clean-results-out, reusing the hardened `executeHttpRequest` outbound path
   (allowlist, outbound rate limit, auth-secret resolution, timeout, response cap). Brave backend today
   (`resolveSearchBackend()`); Tavily/SerpAPI are drop-in branches. Registered via the `lib/app/capabilities.ts`
   app seam ‚Äî promotable to a Sunrise built-in with no interface change. `processesPii = true`.
2. **Research module** (`report/research.ts`) ‚Äî `runReportResearch()` resolves the seeded Report Research
   agent at the reasoning tier and runs a bounded, non-streaming tool loop: each round issues one
   `web_search`, feeds results back, and refines the next query from the accumulated results. Findings are
   the real, deduped grounded URLs (never model-invented); a short synthesis `note` is written per the
   admin's instruction. Best-effort by contract ‚Äî never throws; time-boxed per phase to stay under the
   worker lease.
3. **Generation wiring** (`report/generate.ts`) ‚Äî before/after rounds gated on `research.enabled` + platform
   flag + timing; `before` findings optionally inform the grounded prose as explicitly-general context;
   `before`+`after` findings merged (deduped by URL) onto `content.research` unless `display === 'hidden'`;
   research cost summed into `costUsd`.
4. **Research config tab** ‚Äî per-version `research` block nested in the existing `respondentReport` JSON
   column (no migration), edited from the **Research tab** on the Respondent Report editor (hidden unless
   the platform flag is on). Defensive narrowing in `report/settings.ts`; strict Zod in `authoring/config-schema.ts`.
5. **Rendering** ‚Äî both surfaces read `content.research`: on-screen "Research & sources" table/list
   (`session-complete.tsx`) and PDF (`session-pdf-document.tsx`, `@react-pdf` `Link`).

## Key decisions

- **Best-effort, never fails a report.** A missing agent, unconfigured backend, or provider error returns
  whatever was gathered plus accumulated cost ‚Äî same graceful-degradation contract as the provider-model-audit
  workflow. Until `BRAVE_SEARCH_API_KEY` + the allowlist host are set, the feature is inert.
- **Findings are grounded URLs only** ‚Äî never model-invented; `validateResearch` requires a valid `http(s)`
  URL per finding.
- **No migration** ‚Äî the `research` config and the findings both ride existing JSON columns
  (`respondentReport` config, `AppRespondentReport.content`).
- **Capability lives at the app seam, not forked into the platform** ‚Äî promotable to a Sunrise built-in later.

## Go-live checklist

Feature is inert until all four are done (full detail in `.context/orchestration/report-web-search.md`):

1. `npm run db:seed` ‚Äî applies seed units `069` (flag), `070` (Report Research agent), `071` (`web_search`
   capability + binding).
2. Set `BRAVE_SEARCH_API_KEY` and add `api.search.brave.com` to `ORCHESTRATION_ALLOWED_HOSTS` in `.env.local`.
3. Enable the `APP_QUESTIONNAIRES_REPORT_WEB_SEARCH_ENABLED` feature flag (a `feature_flag` DB row, off by default).
4. Configure on a version's **Respondent Report ‚Üí Research** tab, complete a session, verify the
   "Research & sources" block on the completion screen and in the PDF.

# F2.2 — Tagging

> Committable tracker for **F2.2**, the second feature of **P2 (Admin CRUD over
> questionnaires)**. A per-version **tag vocabulary** plus **M:N assignment to
> questions**, layered onto the F2.1 authoring surface. Downstream consumers are P8
> analytics filtering and F4.1 adaptive selection, so tags need stable identity +
> label; everything else is polish. Sized **~1 PR** (single PR — one coherent review
> pass, one gate-suite run, per the gate-suite-tax calibration).

## Intent

Let an admin define a small **tag vocabulary** on a questionnaire version and assign
those tags to individual questions. The whole surface is gated by
`APP_QUESTIONNAIRES_ENABLED` (404 when off), API-first, and rides the F2.1
mutation pipeline — including the **version-fork-on-launched** lifecycle: editing a
launched version's tags forks a new draft, with the vocabulary and assignments
carried across.

## Decisions (at build)

- **Assignment API = replace-set `PUT`** (`{ tagIds }` replaces a question's whole
  set; idempotent; maps cleanly to a checkbox multiselect) over per-link
  POST/DELETE.
- **A tag = label + optional `color`** from a closed `TAG_COLORS` allowlist
  (`types.ts`); `color` is nullable so it never blocks. Colours are semantic names,
  not hex (the UI maps them to its palette).
- **Same-version invariant is application-layer**, not a DB constraint — the
  assignment route validates every tag id against the question's version (the
  denormalised `AppQuestionSlot.versionId` exists for this). Checked **before**
  forking so a doomed cross-version assignment leaves no orphan draft.
- **Fork copies the vocabulary + re-links assignments.** The F2.1 `// F2.2:` fork
  seam is filled: the deep copy now copies `AppQuestionTag` rows (old→new tag
  id-map) and re-creates `AppQuestionSlotTag` links through the question id-map. The
  `ForkResult` gained a `tagIdMap` so the post-fork assignment `PUT` remaps
  client-sent ids.

## What shipped

**Schema** (`prisma/schema/app-questionnaire.prisma`, migration
`…_app_question_tags`): `AppQuestionTag` (version-scoped; `label`,
`normalizedLabel`, `color?`; `@@unique([versionId, normalizedLabel])`) +
`AppQuestionSlotTag` (M:N join; `@@unique([questionSlotId, tagId])`); both
all-cascade, no `User` FK (UG-1). Migration hand-trimmed of the schema-fold phantom
DDL (3 pgvector DROP INDEX + the `searchVector` ALTER); drift-check 9/9 green.

**Pure core** (`lib/app/questionnaire/tagging/`, Prisma-free): `normalize.ts`
(`normalizeTagLabel`) + `schemas.ts` (`createTagSchema`, `updateTagSchema`,
`setQuestionTagsSchema`). `TAG_COLORS` tuple in `types.ts` (single source).

**Fork seam** (`_lib/fork.ts`): copies the tag vocabulary into the draft and
re-links assignments via the question/tag id-maps; `ForkResult.tagIdMap` added.

**APIs** (`_lib/tagging-routes.ts` helpers — `loadScopedTag`, `asTagConflict`,
`resolveAssignableTagIds`): `POST`/`PATCH`/`DELETE …/versions/:vid/tags[/:tagId]`
(vocabulary) + `PUT …/versions/:vid/questions/:questionId/tags` (replace-set
assignment). Registered under `API.APP.QUESTIONNAIRES`. Audited as
`questionnaire_tag.{create,update,delete,assign}`.

**Read** (`_lib/detail.ts`): `getVersionGraph` extended to load the version
vocabulary + per-question tags in the same single nested query (no N+1); `views.ts`
gains `TagView` + `tags` on `QuestionSlotView`/`VersionGraphView`.

**UI** (`components/admin/questionnaires/`): `tag-vocabulary-editor.tsx`,
`question-tags-editor.tsx` (popover checkbox multiselect → replace-set `PUT`),
`tag-chip.tsx` (shared coloured pill, server + client). Read-only chips on
`version-graph.tsx`. Mutations go through the existing `authoringMutate` /`run`
runner (fork notice + `router.refresh()`).

**Tests:** schema-shape (2 new tables: DMMF + migration SQL — uniques, indexes, FK
cascade, phantom-strip guard); unit (`normalizeTagLabel`, tagging schemas); fork
(`fork.test.ts` extended — vocabulary copy + assignment re-link through the maps);
route integration (`tagging-routes.test.ts` — gate order, scope-404, dedup P2002
400, **cross-version assignment 400 before fork**, replace/clear semantics, fork
remap, audit). `npm run validate` + the full questionnaire suite green.

**Docs:** `questionnaire/schema.md` (tag models) + `questionnaire/admin-ui.md` (tag
endpoints + UI). No CHANGELOG entry — app surface, not the Sunrise platform public
surface (consistent with F1.1 / F2.1).

**Not in F2.2:** extraction-change review (F2.3), re-ingest (F2.4). Tag-aware
analytics filtering (P8) and adaptive selection (F4.1) consume this vocabulary later.

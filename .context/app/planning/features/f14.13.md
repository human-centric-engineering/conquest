# F14.13 — Sessions console

> Committable tracker for **F14.13**. Turns the alpha session-ref browser
> (`/admin/questionnaires/sessions`) from a flat, position-losing list into a self-contained console:
> URL-driven filters + sort, a KPI + charts overview, cohort/round visibility and filtering, and an
> in-list slide-over that shows a session's transcript + report (and regenerates it) WITHOUT navigating
> away.

## Intent

The browser was a single paginated table whose filter/page state lived in React state, so opening a
session (a full-page navigation) always threw the admin back to page 1 — untenable once the list grows.
It also offered only ref-search + status, no overview, and no way to see whether a session belonged to a
cohort/round. This makes the surface usable at scale: filter/scan/open without losing your place.

## Decisions (confirmed with the user)

- **Slide-over drawer**, not full-page navigation, to view a session — the list stays mounted, so
  position is never lost. The full-page viewer stays reachable via an "Open full page" link for deep work.
- **All four chart groups**: sessions-over-time, status + completion distribution, by-client,
  by-questionnaire — a compact, collapsible strip that tracks the active filters.
- **All filter groups**: cohort & round, client & questionnaire, date range + preview toggle, plus
  sortable columns (created, turns).

## What shipped

- **URL-driven state** — filters/sort/page live in the URL (`router.replace(..., { scroll: false })`),
  seeded SSR from `searchParams`. Shareable, back-safe, and position-preserving.
- **Filter bar** (`session-filters.tsx`) — ref search (debounced), status, type (real/preview),
  questionnaire, client, cohort, round (scoped to the chosen cohort), date window, Clear all.
- **Stats strip** (`session-stats.tsx`) — `CqStatTiles` KPIs + recharts (amber-anchored palette).
- **Drawer** (`session-drawer.tsx`) — Transcript tab reuses the read-only `SessionWorkspace`; Report tab
  renders the delivered report (`ReportBody`) + `SessionReportRerun` (history + regenerate). One
  admin-authed fetch (`/admin-view`) seeds both tabs.
- **Cohort/round + client** columns/badges on each row (client via `questionnaire.demoClientId`;
  cohort/round resolved from the session's `roundId`/`cohortMemberId` pointers).
- **Preview sessions hidden by default** — the Type filter defaults to "Real only"; preview (admin
  rehearsal) sessions appear only when explicitly requested ("Preview only" / "All"), flagged with a
  flask symbol. The default `where` excludes them for both the list and the stats/charts.
- **Duration + sittings** — a "Duration" column shows the beginning-to-end span; sessions completed
  across multiple sittings (a >30m gap between turns starts a new one) show a split marker + count, with
  active-vs-elapsed in the tooltip. Derived from turn timestamps (`durationMs`/`activeMs`/`sittings` on
  the list item; also shown in the drawer header). `formatCompactDuration` added to
  `lib/utils/format-duration.ts`.

## Data model

No schema change. Client is the existing `AppQuestionnaire.demoClientId` FK (session→version→
questionnaire). Cohort/round are the plain-String `roundId`/`cohortMemberId` pointers (UG-1), resolved by
batched lookups; the cohort filter pre-resolves the cohort's rounds + members.

## Key files

- Read models: `app/api/v1/app/questionnaire-sessions/_lib/admin-session-list.ts` (extended query +
  shared `buildAdminSessionWhere` + filter-options), `_lib/admin-session-stats.ts` (new).
- Routes: `refs/stats/route.ts` (new), `[id]/admin-view/route.ts` (new; admin-gated transcript + report,
  since the respondent routes are token-gated). All `withAlphaSessionToolsEnabled(withAdminAuth)`.
- UI: `session-ref-browser.tsx` (rewritten orchestrator), `session-filters.tsx`, `session-stats.tsx`,
  `session-drawer.tsx`; `app/admin/questionnaires/sessions/page.tsx` (async `searchParams` seed).
- Client-safe sentinels: `lib/app/questionnaire/admin-session-filter-constants.ts`.

## Out of scope

- Completion-% column sorting (derived post-query; would need raw SQL). Column shown, not sortable.
- De-alpha-ing the surface; cohort-level report generation from this page.

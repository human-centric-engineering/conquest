# F12.2 — AI + document authoring for the intro background

> Committable tracker for **F12.2**, phase 2 of the respondent intro screen. The intro "about this
> questionnaire" background (F12.1) was a plain textarea; this adds three ways to populate it —
> upload a document, generate from a brief, or refine existing text — at both the questionnaire and
> cohort level.

## Intent

Authoring good respondent-facing copy from scratch is friction. Admins already have source material
(a brief, an internal doc) — let them turn it into the intro background in one click, then edit.
Nothing here changes the runtime: the helpers only produce text; the existing config / cohort PATCH
still persists it.

## Decisions

- **Helpers produce text, don't persist** — generate / refine / parse all return a string the field
  populates; the admin reviews and saves through the existing save path. Keeps the surface tiny and
  the review step explicit (AI output is never silently stored).
- **Reuse the composer agent** — the authoring capability binds to the seeded
  `app-questionnaire-composer` agent (same as compose / refine-structure); no new agent.
- **One capability, two modes** — `app_author_intro_background` with `mode: generate | refine`,
  returning `{ background }` via one structured LLM call. Simpler than two near-identical capabilities.
- **Parse is not AI** — document upload just runs `parseDocument` and returns text; only generate /
  refine spend an LLM call.

## What shipped

- **Capability** — `AppAuthorIntroBackgroundCapability` (generate from brief / refine current text
  per instruction → trimmed + capped `{ background }`); registered in `lib/app/capabilities.ts`,
  seeded + bound to the composer agent (`049-author-intro-background-capability`). Pure prompts in
  `intro/authoring-prompt.ts`.
- **Routes** — `POST …/intro-background/author` (generate|refine, dispatches the capability) and
  `POST …/intro-background/parse` (multipart → `parseDocument` → capped text). Both
  `withIntroScreenEnabled` + admin-auth + per-admin sub-cap (compose / ingest limiters). Endpoint
  registry entries added.
- **UI** — `IntroBackgroundField`: a controlled markdown editor with Upload / Generate with AI /
  Refine with AI (popovers), used by the config editor's Intro card and the cohort override form.

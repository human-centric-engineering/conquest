# F9.16 — Agent Settings Evaluation

> Committable tracker for **F9.16**. The questionnaire agents all ship with an empty `model`/`provider`
> and inherit per-task-tier defaults, so an operator had no way to see what each agent actually runs,
> whether it's a sensible cost/quality choice, or to change it — and the app booted with the platform's
> Anthropic-leaning cost-scan defaults. This adds an admin surface to review and tune each agent's
> model/temperature/maxTokens/reasoning-effort against a deterministic OpenAI cost/performance baseline
> (with a hybrid "Explain with AI" layer), and pre-seeds correct OpenAI defaults.

## Intent

ConQuest defaults to **OpenAI**. Give the operator one place to (a) see each agent's resolved model
and settings, (b) compare against a curated recommendation with the cost trade-off, and (c) apply —
model recommendations at the shared task-tier default (inheritance preserved), temperature/effort
per-agent. Ship the app boot-correct for OpenAI rather than relying on a cost-scan that drifts to
Anthropic and throws on embeddings.

## What shipped

- **OpenAI model rows** (`prisma/seeds/009-provider-models.ts`): `gpt-5.5`, `gpt-5.4`, `gpt-5.1`,
  `gpt-5.4-mini`, `gpt-5.4-nano`, `gpt-4.1-nano`, `gpt-4o-transcribe` — blended `costPerMillionTokens`;
  gpt-5 family `paramProfile: 'openai-reasoning'`. Bare floating aliases.
- **Pre-set defaults** (`prisma/seeds/020-orchestration-default-models.ts`): writes
  `AiOrchestrationSettings.defaultModels` (reasoning `gpt-5.4`, chat `gpt-5.4-mini`, routing
  `gpt-4.1-nano`, embeddings `text-embedding-3-small`, audio `gpt-4o-transcribe`) +
  `activeEmbeddingModelId`. **Non-clobbering** — only fills empty slots so operator edits survive.
- **Deterministic engine** (`lib/app/questionnaire/agent-advisory/`): `recommendations.ts` (curated
  table — `TASK_TIER_RECOMMENDATIONS` + 14 `AGENT_RECOMMENDATIONS`, nano `overrideModel` only for the
  three trivial hot-path outliers) and `evaluate.ts` (`evaluateAgentSettings` — resolved model, blended
  `$/M` + per-call estimate + delta, 30-day actuals via `getCostBreakdown`, `isOptimal`, and the
  `temperatureIgnored` caveat for the gpt-5 reasoning param profile).
- **Hybrid "Explain with AI"**: agent `app-agent-settings-advisor` (seed `058-…`), orchestrator
  `explain.ts` (one `runStructuredCompletion`, discriminated result, never throws), schema
  `explain-schema.ts`, prompt `explain-prompt.ts`.
- **API**: `GET …/agent-settings` (admin, flag-gated) returns the evaluation; `POST …/agent-settings/explain`
  (per-admin sub-cap `settingsAdvisorLimiter`, 20/min) returns `{ narrative, suggestion }`. Apply reuses
  the existing `PATCH …/orchestration/settings` (partial-merge tier default) and `PATCH …/orchestration/agents/:id`.
- **UI**: `/admin/questionnaires/agent-settings` page + `AgentSettingsPanel` / `TaskDefaultCard` /
  `AgentSettingCard` (current→recommended comparison, cost delta, actuals, flags, Accept / Apply-all /
  Explain-with-AI). Nav entry in `lib/app/admin-nav.ts`.

## Design notes

- **No new write path / model.** All mutations reuse existing orchestration PATCH endpoints; the
  evaluation is computed in-process and persists nothing.
- **Model at the tier, tuning per-agent.** Respects the inherit-by-default resolver; a model accept
  moves every agent on that tier. Per-agent model pins reserved for the nano outliers.
- **Temperature honesty.** gpt-5 ignores temperature (`openai-reasoning` profile) — surfaced as a flag
  rather than silently tuning a no-op.
- **OpenAI dependency.** Defaults resolve to OpenAI only once an OpenAI provider is active
  (`OPENAI_API_KEY`); provider configs remain operator-managed (no provider seed).

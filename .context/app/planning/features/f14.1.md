# F14.1 — Cohort Report dataset & segmentation foundation

> Committable tracker for **F14.1**, the first feature of P14 (the cross-respondent Cohort Report,
> report kind `cohort`). Lands the analytical substrate everything else consumes — a per-round,
> per-segment dataset built by reusing the F8.1 distribution machinery — plus the platform flag,
> the per-version config block, and the base report model. No UI, no LLM work yet.

## Intent

ConQuest can run a questionnaire across a cohort in rounds and report per respondent, but has no
cross-respondent view. F14.1 builds the foundation for one: a serializable dataset that aggregates a
round's submissions and **segments them by the questionnaire's own demographics** (any `select` /
`number` profile field, plus cohort subgroup), so later features can chart it, narrate it, and score
it. Segmentation is the new capability; everything else (distribution math, k-anonymity) is reused.

## Decisions

- **Round-scoped, version-pinned** — a cohort report is 1:1 with a round (`AppCohortReport.roundId
@unique`), pinned to the version the round delivered. Cross-round rollups are deferred.
- **Reuse the analytics aggregator, don't fork it** — the per-question assembly was extracted from
  `analytics/distributions.ts` into the pure `assembleQuestionDistributions(slots, sessions,
answers)`. `buildCohortDataset` calls it once for the round and again per segment over a session
  subset — so k-anonymity + type-appropriate detail are computed by identical code at every level.
- **k-anonymity per segment, at the data boundary** — a demographic segment below the floor
  (`K_ANONYMITY_THRESHOLD = 5`) has its detail withheld, exactly like the F8.3 version-wide analytics.
  Reused, not re-implemented.
- **Anonymous mode → cohort-level only** — when the version collects no profile, there is no axis to
  split on, so segmentation is empty (subgroups not queried either).
- **Config follows the `respondentReport` precedent** — a lazy `cohortReport` JSON column +
  `narrowCohortReportSettings`, wired through the version config read/write/fork paths so it
  round-trips and survives a launch fork.
- **Flag ANDs cohorts** — `isCohortReportEnabled()` = master AND cohorts AND the cohort-report
  sub-flag (a cohort report only makes sense where rounds exist).

## What shipped

- **Types** — `ReportKind` enum; `CohortReportStatus` / `CohortReportPublishStatus`; the
  `CohortReportLength` / `CohortReportDetailLevel` / `CohortReportFormality` enums; `CohortReportSettings`
  - `DEFAULT_COHORT_REPORT_SETTINGS`; added `cohortReport` to `QuestionnaireConfigShape` +
    `DEFAULT_QUESTIONNAIRE_CONFIG` (`lib/app/questionnaire/types.ts`).
- **Flag** — `APP_QUESTIONNAIRES_COHORT_REPORT_FLAG` constant; `isCohortReportEnabled` /
  `ensureCohortReportEnabled` / `withCohortReportEnabled` gates (`feature-flag.ts`); seed
  `054-cohort-report-flag.ts` (disabled).
- **Schema** — `AppQuestionnaireConfig.cohortReport` JSON column; `AppCohortReport` model (round 1:1,
  status + lease + publishStatus). Migration `…_app_cohort_report` (create-only, phantom pgvector DDL
  stripped per the project convention).
- **Config wiring** — `narrowCohortReportSettings` (`cohort-report/settings.ts`); `cohortReport`
  threaded through `_lib/detail.ts` (CONFIG_SELECT + ConfigRow + toConfigView), the config PATCH
  route, the fork copy (`copy-version-graph.ts`), and the `updateConfigSchema` Zod
  (`authoring/config-schema.ts`).
- **Dataset** — `assembleQuestionDistributions` extracted in `analytics/distributions.ts` (+
  `DISTRIBUTION_SLOT_SELECT` and the `SlotForDistribution` / `SessionForDistribution` /
  `AnswerForDistribution` / `AssembledDistributions` types); `cohort-report/types.ts` (client-safe
  `CohortDataset` / `SegmentDimension` / `CohortSegment` / `CohortSegmentation`);
  `cohort-report/dataset.ts` (`buildCohortDataset` — profile select/number bucketing + subgroup axis +
  anonymous-mode handling); `cohort-report/index.ts` barrel.
- **API** — `GET /api/v1/app/rounds/:id/cohort-report/dataset?versionId=…` (`withAdminAuth`,
  `withCohortReportEnabled`, version-bundled-by-round validation).

## Gates

`isCohortReportEnabled()` (master + cohorts + cohort-report sub-flag) gates the route. The
per-version `config.cohortReport.enabled` toggle becomes the second gate once the admin surface
ships (F14.5).

# F9.13 — Re-evaluate a chat by its support reference

> Committable tracker for **F9.13**. Gives every chat a short, user-facing **support reference**, and
> makes a real conversation re-evaluable after the fact: inspector traces are now persisted for every
> session, and an admin can look a chat up by its reference and run the turn evaluator over the exact
> calls a turn made. Before this, only live preview turns could be evaluated, and there was no way for
> a respondent to point support at their conversation.

## Intent

A respondent who has a bad experience can quote a reference ("Ref: 7F3K-9M2P"); an admin uses it to
open the conversation and re-evaluate any turn against its saved traces — feeding the same
persisted-verdict + flag-for-learning pipeline as F9.12.

## Decisions (confirmed with the user)

- **Always capture** inspector traces for real sessions (not opt-in) — the simplest end-to-end; the
  user accepted storing prompts + respondent PII on every turn for this.
- **Ref placement: footer + completion** — a quiet, always-visible chip during the chat and on the
  completion screen (copy-on-click).
- **Ref format: short readable code** — 8-char Crockford base32 (no I/L/O/U), grouped `XXXX-XXXX`.

## What shipped

**Phase A — session ref.** `AppQuestionnaireSession.publicRef` (nullable `@unique`) + migration
(hand-authored — a unique constraint trips `migrate dev`'s interactive confirm). `session-ref.ts`:
generate / format (group) / normalize (fold O→0, I/L→1, strip grouping). Generated in all six
session-create paths. Threaded `buildTurnContext` → `SessionStatusView.ref` → the lifecycle strip
(`SessionRefChip`, copy-on-click) + the completion screen.

**Phase B — persist traces.** `AppQuestionnaireTurn.inspectorCalls` Json + migration. Capture is now
unconditional in the messages route (decoupled from the preview gate); SSE **emission** of the live
frame stays preview-gated. Threaded messages route → `persistTurn` → `recordTurn`. No read path
surfaces `inspectorCalls` to respondents (transcript/resume select only message/warnings/reasoning).

**Phase C — lookup + re-evaluate.** Shared `inspector/schema.ts` (Zod dump validator, used by the
live route too — refactored). Shared `turn-evaluation-context.ts` (`loadTurnEvaluatorAgent`,
`buildObjectivesContext`) extracted from the live route. `lookupSessionByRef` read model +
`GET …/turn-evaluations/by-ref/:ref`. `runSavedTurnEvaluation` orchestration +
`POST …/questionnaire-sessions/:id/turns/:ordinal/evaluate-saved` (admin, rate-limited, **not**
preview-gated). Admin `RefLookupPanel` on the turn-evaluations page: look up → list turns → Evaluate
(saved) per turn → inline score.

## Gotchas

- **Ref capture vs emission** are now decoupled: traces are persisted for ALL sessions, but the live
  `inspector` SSE frame is still emitted only for preview + toggle. A real respondent never sees it.
- **PII**: `inspectorCalls` holds prompts + respondent words. It is admin-only (no respondent read
  path selects it) and cascades/erases with the turn row, but it does materially increase what a real
  session stores — the deliberate cost of "always capture".
- **Saved dump is validated** at the read seam with the same Zod schema as the live POST — a
  pre-column or malformed turn surfaces as `no_traces` (422), not a 500.
- **Migrations**: both columns added via hand-authored SQL (unique-constraint confirm / phantom
  pgvector drops) — `publicRef` nullable-unique (Postgres allows many NULLs), `inspectorCalls` JSONB
  default `'[]'`.

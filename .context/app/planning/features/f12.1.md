# F12.1 — Respondent intro / splash screen

> Committable tracker for **F12.1**. An admin opt-in cover screen shown before the questionnaire
> starts: it introduces the process (adapting to the presentation mode), explains what the respondent
> gets at the end (adapting to the respondent-report settings), and shows an admin-authored
> "about this questionnaire" background section — optionally overridden per cohort. The respondent
> presses a button to begin.

## Intent

A respondent currently lands straight in the conversation, with the welcome as the first chat turn
and an immediate LLM kickoff. There was no screen to set expectations: what kind of experience this
is, what they'll receive at the end, and context about who's asking and why. This adds that screen,
gated so existing questionnaires are untouched (off by default).

Built as **phase 1**: the splash + a plain-text/markdown background field + the cohort override.
Phase 2 (deferred) adds richer background authoring — document upload, generate-with-AI, refine-with-AI
— reusing the existing `parseDocument` + compose/refine capability patterns.

## Decisions

- **Admin opt-in, default off** — a per-version `config.intro.enabled` toggle plus the platform flag;
  existing launched versions are unchanged.
- **Copy is derived, not stored** — only the background + button label are authored; the
  how-it-works / what-you'll-get copy is computed from `presentationMode` + `respondentReport` at
  runtime so it can never drift from the live settings.
- **Cohort override = replace** — a cohort's `introBackground`, when set, replaces the version-level
  background for that cohort's respondents; blank inherits. Mirrors the theming fallback.
- **Splash gates the workspace mount** — the workspace fires the LLM kickoff on mount, so the splash
  sits _before_ it (`SessionEntry`); the kickoff is deferred until the respondent presses begin.

  schemas; config read view + fork copy carry `intro`.
- **API** — `GET /api/v1/app/questionnaire-sessions/:id/intro` (token-authed resolved intro for the

- **Respondent UI** — `SessionEntry` gate + `QuestionnaireSplash` (brand-aware, adaptive sections,
  proceed button); wired into the authenticated page (server-resolved) and the anonymous boot
  (client fetch, fresh-session only).
- **Admin UI** — Intro screen card on the Settings tab (gated on the `introScreen` workspace flag);
  cohort "Intro background override" field.
- **Flag** — `APP_QUESTIONNAIRES_INTRO_SCREEN_FLAG` + `isIntroScreenEnabled()`; seed
  `048-intro-screen-flag` (disabled by default).

## Gates

`isIntroScreenEnabled()` = master `APP_QUESTIONNAIRES_ENABLED` AND
`APP_QUESTIONNAIRES_INTRO_SCREEN_ENABLED`. Second gate: per-version `config.intro.enabled`. Third:
fresh session only (`autoStart`) — resume skips the splash. All off → today's behaviour.

# F9.11 — Turn evaluation

> Committable tracker for **F9.11**. Adds an admin-only **interview-quality evaluator** that runs
> from the Preview Turn Inspector over **one completed turn** and returns a scored, sectioned verdict.
> Before this, the inspector only _described_ a turn (the raw agent calls); there was no way to _judge_
> whether the turn behaved well or generated useful information.

## Intent

Give developers/researchers/prompt-engineers a one-click "is this turn any good?" verdict: instruction
compliance, interviewing quality, extraction quality, question-selection quality, information gain,
missed opportunities, prompt drift, and cost/efficiency — judged against the prompts that produced each
call, not from outputs alone. Never shown to respondents.

## Decisions (confirmed with the user)

- **Output: hybrid** — typed/enum'd headline scores (overall 0–100, per-call 0–100, interviewer 1–10,
  ratings) + per-section markdown prose. Machine-usable for chips/trends, robust to validate.
- **Ephemeral** — no DB. Inspector data is itself live-only, so the verdict renders in the drawer and
  is discarded on close (the F5.1 `evaluate-preview` posture). Persistence is a clean later follow-up.
- **Seeded configurable judge agent** (`turn-evaluator`, `kind: 'judge'`) for the model binding —
  admin-tunable, empty provider/model → system default. Same pattern as the design-evaluation judges.
- **Copy + Download** on the verdict (user-requested) — both emit the same serialized Markdown.

## Key architectural fact

Inspector data is **live-only and never persisted** — the server streams `inspector` SSE frames
(full `TurnInspectorData` dumps) only for an admin preview session with the toggle on, and the client
holds them in `inspectorTurns`. So the dump already exists client-side: the drawer POSTs it to the
route, which runs one structured LLM call and returns the verdict. No persistence, no prompt
reconstruction. The route reloads the questionnaire **objectives** (goal/audience/strategy/tone) by
session id so they can't be spoofed.

## What shipped

**Core — pure module `lib/app/questionnaire/turn-evaluation/`**

- `schema.ts` — the hybrid Zod verdict contract + `z.toJSONSchema` export, `validateTurnEvaluation`,
  `buildTurnEvaluatorRetryMessage`. Headline scores/ratings typed; prose sections bounded markdown
  strings.
- `types.ts` — `TurnEvaluationInput` (the dump + optional `TurnEvaluationContext`).
- `prompt.ts` — `buildTurnEvaluatorPrompt`: the load-bearing system rubric (only-calls-that-ran,
  judge-against-the-prompt, JSON-only) + a user message that reuses `formatInspectorTurn` to serialize
  the dump and weaves in the context block (degrading when absent).
- `serialize.ts` — `serializeTurnEvaluation`: the verdict → Markdown doc following the spec headings,
  the single source of truth for Copy + Download.
- `evaluate-turn.ts` — `evaluateTurn(input, agent, opts)`: resolve binding (`reasoning` tier) →
  `runStructuredCompletion` (parse → retry-at-temp-0 → cost-sum) → fire-and-forget `logCost`. A plain
  service, not a `BaseCapability` (single call, no fan-out).
- `index.ts` — barrel.

**Route**

- `POST /api/v1/app/questionnaire-sessions/:id/evaluate-turn` — `withTurnEvaluationEnabled`
  (gate before auth) → `withAdminAuth` → per-admin `turnEvaluationLimiter` (20/min) → validate the
  client dump (Zod) → load session (preview-only) + version objectives → load `turn-evaluator` agent →
  `evaluateTurn` → `{ verdict, costUsd, model }`. Fail-soft: a thrown evaluator returns a 502 envelope,
  never a 500 that breaks the drawer. Endpoint helper `evaluateTurn(id)`.

**Flag + resolver**

- Constant `APP_QUESTIONNAIRES_TURN_EVALUATION_FLAG`; `isTurnEvaluationEnabled` +
  `ensure/withTurnEvaluationEnabled` in `feature-flag.ts` (master AND sub-flag, mirror of design-eval).

**Seeds**

- `042-turn-evaluation-flag.ts` (disabled by default) and `043-turn-evaluator-agent.ts` (one judge
  agent, empty provider/model, low temperature, 4096 max tokens). No migration (existing `AiAgent` table).

**UI**

- `turn-inspector-drawer.tsx` — new `sessionId` prop (threaded from `questionnaire-chat.tsx`); a
  per-turn **Evaluate turn** button; `TurnEvaluationSection`/`TurnEvaluationPanel` render the verdict
  (headline chips + interviewer sub-score grid + full markdown via `MarkdownContent`) with **Copy**,
  **Download** (`turn-{n}-evaluation.md`), and **Re-run**.

## Gotchas

- The dump is **client-supplied** (inspector data is never persisted) — validate it with Zod, never
  trust shape. Objectives are loaded **server-side** by session id.
- Gated **twice**: master+sub flag AND preview-only session — the same envelope the inspector that
  produces the dump runs under, so the evaluator can only run where the inspector runs.
- The verdict is **ephemeral** — it lives in component state; closing the drawer discards it.
- Copy and Download share `serializeTurnEvaluation`, so the clipboard, the `.md`, and the on-screen
  body never drift.

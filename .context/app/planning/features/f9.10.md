# F9.10 — Requiredness controls across create / import / edit

> Committable tracker for **F9.10**. Gives the admin a deliberate requiredness decision at each of
> the three authoring seams. Before this, every question landed **optional** (`required` hardcoded
> `false` in `writeGraph`), the document's own required markers were dropped, and there was no bulk
> lever — the only control was the per-question switch.

## Intent

A real-world questionnaire usually wants most fields mandatory, and an uploaded document often states
which fields are required. The product needed: a "make everything required" default at create/import
(checked by default), a way to honour the document's markers instead, and a one-click bulk toggle in
the editor.

## Decisions (confirmed with the user)

- **Import → two-mode radio** (not a single checkbox): **Make all fields required** (default) vs **Use
  the document's required markers**. The second mode is clearer about its effect — only flagged fields
  become required, the rest stay optional.
- **Edit → tri-state checkbox** "All questions required" (checked / unchecked / indeterminate-mixed),
  toggling bulk-sets every question. Closest to "a checkbox that changes all by bulk".
- **Create (compose) → single checkbox** "Mark all questions as required", checked by default. The
  composer infers no requiredness, so there is no "from source" mode — it's a pure toggle.
- **Default everywhere is "all required"** — the checked-by-default behaviour the user asked for.

## What shipped

**Core — one policy on the persist writer**

- `RequirednessPolicy = 'all' | 'optional' | 'source'` + `resolveRequired()` in `_lib/persist.ts`.
  `writeGraph(tx, versionId, extraction, requiredness = 'optional')` resolves each slot's `required`
  from the policy; `persistIngestion` forwards its new `requiredness` input (defaulting to `'all'`).
  `writeGraph`'s own default stays `'optional'`, so the refine path (`replaceVersionStructure`) is
  untouched.
- `extractedQuestionSchema.required: z.boolean().optional()` (`ingestion/extraction-schema.ts`) — the
  per-question signal the **`'source'`** mode reads. Flows through `ExtractedQuestion` →
  `ExtractQuestionnaireStructureData.questions` with no capability change. The extractor prompt
  (`ingestion/extraction-prompt.ts`) instructs `required: true` **only** when the source explicitly
  marks a field mandatory (asterisk, `(required)`, `mandatory`, a Required column) — a faithful read,
  not a judgement call, and no change record.

**Import seam**

- `upload-input.ts` — `requiredMode: 'all' | 'source'` (default `'all'`; unrecognised value → 400).
  `route.ts` passes `requiredness: adminMeta.requiredMode` into `persistIngestion`.
- `upload-questionnaire-dialog.tsx` — a native two-radio fieldset (no `radio-group` component exists),
  default `all`, sent as a `requiredMode` FormData field.

**Create seam**

- `compose-input.ts` — `requiredAll: z.boolean().optional()` on `composeRequestSchema`. Both
  `compose/route.ts` and `compose/stream/route.ts` map `requiredAll === false ? 'optional' : 'all'`
  (omitted ⇒ `'all'`).
- `compose-studio.tsx` — a `Checkbox` "Mark all questions as required" (default checked), `requiredAll`
  added to the stream request body.

**Edit seam**

- `bulkSetRequiredSchema = z.object({ required: z.boolean() })` in `authoring/schemas.ts` (barrelled).
- New route `PATCH /api/v1/app/questionnaires/[id]/versions/[vid]/questions` — forks a launched version
  (`forkVersionIfLaunched`), `updateMany({ where: { versionId: editId }, data: { required } })`, audit
  `questionnaire_question.bulk_required`, returns `{ updated, required }` + `forkMeta`. Endpoint helper
  `versionQuestions(id, vid)`.
- `version-editor.tsx` — a tri-state `Checkbox`: `checked` when all questions required, `indeterminate`
  (set on the native input via a `ref`, since the project's `Checkbox` has no such prop) when mixed.
  Toggling routes through the existing `run` runner (fork notice + refetch + busy-lock for free).

## Tests

- `persist.test.ts` — the three policies (all / optional / source) + the new `'all'` default; existing
  slot-mapping assertion updated `required: false → true`.
- `extraction-schema.test.ts` (ingestion) — optional `required` accepted (true/false/omitted), non-boolean rejected at `questions.N.required`.
- `upload-input.test.ts` — `requiredMode` default `'all'`, explicit `'source'`/`'all'`, garbage → 400.
- `bulk-required-route.test.ts` (new) — gate-off 404 before auth, 401/403, scope-404, happy path
  `updateMany`, fork retarget to the new draft, body validation 400.
- `compose-routes.test.ts` — both routes thread `requiredness` (`'all'` default, `'optional'` on `requiredAll:false`).

## Gotchas

- The project `Checkbox` is a native-input wrapper with **no `indeterminate` prop** — the mixed state
  is set imperatively via a `ref` effect.
- There is **no `radio-group` UI component** — the import control is built from native `<input type="radio">`.
- The bulk editor checkbox writes `required` directly via `updateMany`; it does **not** go through
  `RequirednessPolicy` (that policy is for the create/import write path only).

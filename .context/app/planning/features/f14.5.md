# F14.5 — Cohort Report settings, templates & block editor

> Committable tracker for **F14.5**. The authored-report experience: per-version generation settings
>
> - structure template, and a full WYSIWYG editor for the generated report — rich-text sections,
>   reorder, add/delete/duplicate, and per-section AI-assist — each save a version-controlled `admin`
>   revision.

## Intent

Generation (F14.3) gets a report most of the way; F14.5 lets the admin shape it: tune how it's
generated (length/detail/formality/structure/context/scoring), then edit the result directly — rich
text, reorder sections, rewrite a section with AI help — without losing the version history.

## Decisions

- **Full WYSIWYG (Tiptap), HTML storage** — the user chose a true block/WYSIWYG editor over a
  markdown+drag surface. Section bodies are stored as **HTML** so the editor, read view and PDF speak
  one format; the AI's markdown is converted once at the generation boundary (`marked`), and
  `CohortReportSection.format` marks html vs legacy markdown.
- **Sanitise at the render boundary** — stored HTML is sanitised with dompurify (explicit tag
  allowlist) in `CohortSectionBody`, the standard XSS defence; the editor keeps raw HTML editable.
- **Edits are revisions** — `PATCH …/cohort-report` validates the full content and appends an `admin`
  revision (never mutates); AI-assist (`POST …/refine`) returns a section the editor drops in, saved
  on the next PATCH.
- **Reorder via up/down, not drag** — robust + dependency-free; satisfies "move sections" without a
  DnD library (the WYSIWYG editing was the headline the user picked).
- **Settings on the Scoring tab** — per `settings-on-settings-tab`, cohort-report config is run-time
  config; it shares the per-version, cohort-report-gated **Scoring** tab with the scoring builder
  (both are per-version cohort-report configuration), avoiding surgery on the large config editor.
- **Structure template = the `structure` config field** — admin-authored outline the agent fills;
  no separate template-library model in v1.

## What shipped

- **Deps** — `@tiptap/react` + `@tiptap/starter-kit` + `@tiptap/pm` (v3, React 19); `marked` (md→html).
- **Content** — `CohortReportSection.format` (`markdown`|`html`); `validateCohortReportContent`
  carries it; `richtext.ts` (`markdownToHtml`); generation converts summary + section bodies to HTML.
- **AI-assist** — `refine.ts` (`refineCohortReportSection`) + `POST …/cohort-report/refine` (sub-cap).
- **Edit** — `PATCH …/cohort-report` (validate full content → append `admin` revision, audit).
- **UI** — `RichTextEditor` (Tiptap), `CohortSectionBody` (sanitised HTML / markdown fallback),
  `CohortReportEditor` (sections: rich-text, reorder, add/delete/duplicate, AI-assist; recommendations
  - actions), `CohortReportSettingsForm`; round panel gains an Edit toggle; Scoring tab hosts the
    settings form above the builder.
- **Endpoints** — `cohortReportRefine` helper; PATCH on the cohort-report endpoint.

## Gates

`isCohortReportEnabled()` gates the refine + edit routes; the Scoring tab (`flags.cohortReport`) hosts
the settings + builder. Generation/edit still require the per-version `enabled` toggle.

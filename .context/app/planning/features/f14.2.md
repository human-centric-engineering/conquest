# F14.2 — Cohort Report charts

> Committable tracker for **F14.2**. A declarative chart layer over the F14.1 dataset: a `ChartSpec`
> says what to plot, a pure builder turns it into uniform `ChartData`, and a recharts component
> renders it. One `ChartData` is designed to drive both the web chart and the F14.6 PDF chart, so
> they can never diverge.

## Intent

The cohort report needs charts — distributions, by-segment comparisons, completion, segment sizes —
that the AI can propose and the admin can curate, and that render identically on screen and in the
downloaded PDF. F14.2 builds that as a **declarative spec + a single pure series builder**, so the
chart-selection logic (F14.3/F14.5) and the two renderers (web now, PDF in F14.6) all share one
contract.

## Decisions

- **Declarative `ChartSpec`, pure `buildChartData`** — the spec carries no rendering state; the
  builder resolves it against a `CohortDataset` into a uniform `ChartData` (categories × series).
  Pure → unit-tested directly and reused by the PDF renderer (the "shared series layer" the plan
  flagged as what makes react-pdf charts tractable, since recharts is DOM-only).
- **Five kinds cover the round** — `question_distribution`, `question_mean_by_segment`,
  `response_rate_by_segment`, `completion_by_segment`, `segment_sizes`. Enough for the AI to express
  "seniority shows the biggest spread → bar chart" without an open-ended chart DSL.
- **Degrade, never mislead** — a stale/unknown reference or free-text question → `empty: true`; a
  k-anonymity-suppressed question/segment → `suppressed: true` or omission, never a fake zero.
- **Presentational component** — `CohortChart` takes built `ChartData`, so the server (or a parent)
  owns the data and the component just draws; matches the evaluation/cost trend-chart precedent.

## What shipped

- **Contract** — `cohort-report/chart-types.ts`: `ChartSpec`, `ChartData`, `ChartDatum`,
  `ChartSeriesDef`, the `COHORT_CHART_KINDS` / `COHORT_CHART_DISPLAYS` tuples + types.
- **Builder** — `cohort-report/chart-series.ts`: `buildChartData(spec, dataset)` (distribution
  buckets per question type; mean / response-rate / completion / size per segment; suppression +
  empty handling). Barrel updated.
- **Component** — `components/admin/questionnaires/cohort-report/charts/cohort-chart.tsx`: recharts
  `BarChart` (single / grouped / stacked), percent-aware axis + tooltip, `role="img"`, suppressed +
  empty placeholders. `'use client'`.

# F13.3 — Round Phases

> Committable tracker for **F13.3**. A round can stagger access by cohort **subgroup**, so one group
> (e.g. the Senior Leadership Team) takes it before the rest — useful for a pilot ahead of a wider
> rollout, and for letting an early group SEED [Learning Mode](./f13.2.md) for those who follow. Off
> by default behind `APP_QUESTIONNAIRES_ROUND_PHASES_ENABLED`; a round with no phases behaves exactly
> as before.

## Intent

Organisations rarely want everyone to start at once. A leadership team may want to try ConQuest
first; or, in a learning-enabled round, a senior group goes first specifically so the interviewer can
carry their (anonymised, generalised) context into the rest of the org's conversations. F13.3 makes a
round's access window vary by subgroup, with each subgroup's start (and optionally end) staggered
inside the round window.

## Decisions (confirmed with the user)

- **Reusable cohort subgroups** — a subgroup (`AppCohortSubgroup`) is defined once on the cohort and
  carried across rounds; a round attaches a window to it via `AppRoundPhase`. A member belongs to 0–1
  subgroup (`AppCohortMember.subgroupId`, `onDelete: SetNull`).
- **Nested windows** — a phase window must sit INSIDE the round window (validated at the admin seam);
  the round window is the hard outer cap for everyone.
- **Per-phase end mode** — `hard` (members lose access at the phase close) or `relaxed` (the phase
  close is only a target — used for staggered notifications; access continues to the round close). The
  phase OPEN is always enforced; staggering the start is the point.
- **Effective window = round, narrowed by the member's phase** — members with no subgroup, or whose
  subgroup has no phase on this round, fall back to the round window. Single source of truth:
  `resolveEffectiveWindow` (`lib/app/questionnaire/rounds/phases.ts`).
- **Staggered send, given the token model** — round invites store only a token HASH; the plaintext
  token exists only at generation. So "sending" is folded into generation: per-phase send generates +
  emails just that subgroup's freshly-minted links; the auto-stagger hook generates+sends each phase
  when its window opens. Idempotent, so a phase is effectively dispatched once.
- **Learning Mode interplay needs no new code** — the digest is already round-scoped and rebuilds on
  each completion, so an early phase naturally seeds later ones.

## What shipped (by build phase)

- **A — Subgroups** — `AppCohortSubgroup` + `AppCohortMember.subgroupId`; subgroup CRUD under
  `/api/v1/app/cohorts/:id/subgroups`; member PATCH accepts `subgroupId` (validated same-cohort);
  cohort read models expose subgroups + per-member `subgroupId`; admin Subgroups panel + per-member
  selector; flag `APP_QUESTIONNAIRES_ROUND_PHASES_ENABLED` (seed `053`) + `withRoundPhasesEnabled`.
- **B — Phases** — `AppRoundPhase` (window + `endMode`); pure `phases.ts`
  (`validatePhaseWindowNesting` + `resolveEffectiveWindow`); phase CRUD under
  `/api/v1/app/rounds/:id/phases` with nesting + cohort checks; round detail exposes phases; admin
  Phases panel (add/edit/delete, end-mode select).
- **C — Enforcement** — `evaluateRoundAccess` takes the member's phase and checks the effective
  window, with `PHASE_NOT_YET_OPEN` / `PHASE_WINDOW_CLOSED` codes; `assertRoundAccess` loads the
  member's subgroup + phase (flag-gated); session snapshots `cohortSubgroupId` at create; per-member
  invite expiry pins to the member's effective close.
- **D — Stats** — `sessionCountsBySubgroup` (one grouped query, keyed by the session snapshot);
  per-phase completion on `RoundPhaseView`; Completion column in the Phases panel.
- **E — Delivery** — `sendRoundInvitationEmail` (frictionless URL); generator gains `{ send,
subgroupId }`; per-phase "Send invites" button + `POST …/phases/:phaseId/send-invites`; auto-stagger
  via `dispatchDuePhaseInvitations` + the app-owned `POST …/rounds/maintenance/dispatch-phase-invites`
  hook (point a scheduled workflow / cron at it — no platform tick fork).

## Gates

`isRoundPhasesEnabled()` (master + cohorts + round-phases sub-flag) gates the subgroup/phase
authoring surfaces and the access/expiry phase lookups. With the flag off, the panels 404/hide and the
access guard falls back to the round window for everyone — today's behaviour.

# F8.7 — Respondent profile capture (form gate + validation + conversational)

> Committable tracker for **F8.7**. Extends F8.3's profile capture into a first-class feature: a
> **blocking in-flow form gate** (the default), an alternative **conversational** collection mode, and
> **per-field validation** (deterministic / agentic / hybrid). Gated by `APP_QUESTIONNAIRES_ENABLED`
> (no dedicated capture flag — purely per-version config).

## Intent

F8.3 collected `profileFields` via a **pre-session** form (`ProfileStartForm`) on the `/start` page,
authed-invitation path only — the public/no-login path never collected them. This feature makes
capture a real in-flow step and adds validation:

1. **Form gate (default).** A standard form rides the carousel **after the intro, before the chat**,
   blocking progress (and the opening LLM turn) until filled + validated.
2. **Conversational mode.** The interviewer gathers the fields naturally in-chat; a best-effort pass
   persists them once complete.
3. **Per-field validation.** Deterministic (Zod/regex), agentic (LLM tidy + garbage-flag), or hybrid.

## Decisions (confirmed with the user)

- **Both modes now; `captureMode` default `form`.** Conversational is the lighter, deferrable second
  way — built, but the form gate is the primary deliverable.
- **Gate keys off `anonymousMode`, NOT authed-vs-public.** A public no-login link CAN collect a name
  (admin's choice); an anonymous link stays PII-free with no gate/directive/snapshot. `resolveSessionCapture`
  owns this — moved OUT of `start-context.ts`. Invariant enforced in three places (resolver null,
  server PUT reject, workspace `showCapture` false); the PUT is the true boundary.
- **Reuse `profileFields`.** Extend `ProfileFieldConfig` with a per-field `validation` mode (in the
  existing JSON — no column); add scalar `captureMode` (one column, one migration).
- **Agentic validation is non-fatal.** An LLM outage falls back to the deterministic value — never
  blocks a respondent (mirrors `resolveAnswerFit`).
- **Supersede the pre-session form.** `ProfileStartForm` + `start-context.ts` deleted; the `/start`
  page just creates/resumes (idempotent) and redirects. `buildFormSchema` moved to `profile/form-schema.ts`.

## Build shape

- **Schema / types** — `types.ts`: `PROFILE_FIELD_VALIDATION_MODES`, `CAPTURE_MODES`,
  `ProfileFieldConfig.validation`, config `captureMode` (+ default). `config-schema.ts`: per-field
  `validation` (optional-with-default so legacy JSON parses) + `captureMode`. Prisma: `captureMode`
  column (migration `20260715120000_app_questionnaire_capture_mode`, hand-authored ADD COLUMN, phantom
  pgvector DROPs avoided; 5 vector indexes verified intact). `detail.ts`: `asCaptureMode` narrower +
  `asProfileFields` routed through `parseProfileFields` (applies the validation default).
- **Validation service** — `profile/validate-profile-fields.ts`: `validateProfileSubmission` composing
  deterministic/agentic/hybrid, batched `runStructuredCompletion` (chat tier), non-fatal, cost-logged.
- **Runtime endpoint** — `questionnaire-sessions/[id]/profile/route.ts` (GET config+satisfied, PUT
  validate+upsert), `resolveTurnAccess`, anonymous guard, `withLiveSessionsEnabled`. Shared
  `profile/profile-snapshot.ts` (idempotent upsert) + `profile/resolve-capture.ts` (`resolveSessionCapture`).
- **Carousel gate** — `session-workspace.tsx`: new blocking `capture` view (intro → capture → persona
  → chat), kickoff deferred until `captureDone`, forward-nav lock + toggle suppression while blocking.
  Surface `profile/profile-capture-gate.tsx` (RHF + Zod + `FieldHelp`, PUTs, maps `fieldErrors`).
- **Seeding** — authed `[sessionId]/page.tsx` SSR-resolves `capture`; `anonymous-session-boot.tsx`
  `fetchCapture` (fails soft to null).
- **Admin UI** — `config-editor.tsx`: "Respondent profile fields" group gains a `captureMode` control
  - a per-field validation `Select`, each with `FieldHelp`.
- **Conversational** — `profile/conversational-capture.ts`: `buildProfileCaptureInstructions` injected
  into the interviewer prompt (`question-stream.ts` `profileCapture` section) via `messages/route.ts`
  when conversational + non-anonymous + no snapshot; `extractAndPersistConversationalProfile` best-effort
  post-turn extraction → validate → upsert (only a COMPLETE profile is written, so the directive stops).

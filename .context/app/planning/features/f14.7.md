# F14.7 — Data slots as a first-class cohort analysis dimension

> Committable tracker for **F14.7**. The original brief named the thematic analysis as being over
> "the recorded answers (**data slots** and aggregated questionnaire data)". F14.1–F14.6 aggregated
> question answers but not data-slot fills; this makes data slots — the semantic substance of the
> responses — first-class in the cohort dataset, the charts, and (critically) the narrative agent's
> prompt.

## Intent

Data slots hold the agent's natural-language restatement of each respondent's _position_ per topic —
the meat of what people actually said, beyond typed question values. The cohort report's thematic
analysis is only as good as that material, so the narrative agent must see it (synthesised, not
quoted), and the admin must be able to chart slot engagement.

## Decisions

- **Aggregate counts on the client dataset; raw paraphrases server-side only.** `CohortDataset.dataSlots`
  carries fill rate / confidence / provenance (overall + per segment) — k-anonymity-safe, chartable.
  The raw respondent paraphrases that feed thematic analysis are loaded separately
  (`data-slot-material.ts`) straight into the agent prompt and **never** returned to the browser.
- **Synthesise, never quote.** The prompt instructs the agent to produce anonymised themes/tensions
  and forbids verbatim quotes or attribution.
- **k-anonymity at two levels.** The whole material block is skipped for a below-floor cohort, and a
  single slot answered by fewer than the threshold contributes no samples.
- **Reuse the existing groupings + chart contract.** Per-segment fill rates use the same
  `DimensionGrouping`s as question segmentation; two new chart kinds (`dataslot_response_overall`,
  `dataslot_response_by_segment`) plug into the F14.2 `ChartData` pipeline.

## What shipped

- **Types** — `CohortDataSlots` / `CohortDataSlotSummary` / `CohortDataSlotByDimension` (+ a
  `dataSlots?` field on `CohortDataset`).
- **Aggregation** — `buildDataSlots` in `dataset.ts` (per-slot fill/confidence/provenance overall +
  per-segment fill rate, suppressed; undefined when no slots/fills).
- **Agent material** — `data-slot-material.ts` (`buildDataSlotThemeMaterial`, server-only, capped +
  k-anonymity-gated); wired into `generate.ts` as a "RESPONDENT POSITIONS BY DATA SLOT" prompt section
  (gated on a non-suppressed cohort).
- **Digest + catalog** — `buildCohortDatasetDigest` adds a data-slot summary section;
  `buildChartCatalogText` lists data-slot keys.
- **Charts** — `dataslot_response_overall` + `dataslot_response_by_segment` kinds + `ChartSpec.dataSlotKey`;
  `buildChartData` handles them; generation prompt advertises them.

## Tests

- `cohort-report/dataset.test.ts` — data-slot overall + per-segment aggregation; omitted when no fills.
- `cohort-report/data-slot-material.test.ts` — per-slot block, paraphrase-over-value, per-slot k-anonymity floor.
- `cohort-report/chart-series.test.ts` — the two data-slot chart kinds.
- `cohort-report/content.test.ts` — digest data-slot section.
- `cohort-report/generate.test.ts` — mocks extended for the session/data-slot reads.

## Gates

Same as the rest of the cohort report (`isCohortReportEnabled()`); the data-slot material only loads
when the version actually has data slots with fills and the cohort clears the k-anonymity floor.

# F7.5 — Suggestive brand chrome

> Committable tracker for **F7.5**, a P7 theming enhancement. F3.4 gave a demo client four
> brand fields (CTA / accent colour, logo, welcome copy) and F7.1 applied them to the chat
> surface. F7.5 widens that vocabulary so the respondent session can _suggest_ a client's
> brand — a deep surface header band, a gradient CTA, and a backdrop for logos drawn to sit
> on one — while staying generic enough for any client to arrange differently. Triggered by
> the first live demo (Merlin5, slug `test-demo`).

## Intent

The Merlin5 brand is a deep aubergine (`#280039`) with a magenta accent (`#FF03DF`) and a
logo built to sit on the dark surface — so the F3.4 four-field model under-served it: the
purple read as a "CTA colour", and a logo on white looked broken. F7.5 adds the missing
chrome cues so the question session (and the admin "Preview as respondent") reads as _that_
brand, not a recoloured default — without trying to clone the client's marketing site.

## Decisions

- **Suggestive, not a pixel-match.** A surface band + gradient CTA + logo backdrop evoke the
  brand; we deliberately don't replicate site layout.
- **Configurable for other arrangements.** Every new field is independent and optional, so a
  light-surface / solid-CTA / no-backdrop client is just as expressible.
- **The logo backdrop is a checkbox.** One of a brand's colours is often the logo's intended
  background. `logoBackgroundEnabled` (the requested toggle) paints a solid colour behind the
  logo; the colour defaults to the surface colour when left blank.
- **Email gets the safe subset.** Gradients / surface bands don't render reliably across mail
  clients, so the invitation email keeps its solid CTA but does paint the (solid) logo
  backdrop — the one cue that meaningfully fixes a dark-brand logo.

## What shipped

### Schema (`AppDemoClient`, migration `…_app_demo_client_session_chrome`)

Four new columns, all back-compatible (nullable / `default false`):
`surfaceColor`, `ctaColorEnd`, `logoBackgroundColor`, `logoBackgroundEnabled`. Migration was
generated `--create-only` and stripped of the phantom pgvector `DROP INDEX` / `DROP DEFAULT`
diff before applying (the standing app-migration gotcha).

### Theming module (`lib/app/questionnaire/theming/**`, Prisma-free)

- `DemoClientTheme` gains the four fields as **optional** (the original four stay required) —
  `resolveTheme` reads them defensively (`?? null` / `?? false`), so absent === null and older
  selects / forks / tests need no change.
- `ResolvedTheme` gains `surfaceColor`, `ctaColorEnd`, and a **pre-resolved** `logoBackgroundColor`
  (null whenever the toggle is off; otherwise the explicit colour falling back to `surfaceColor`).
- `themeToCssVariables` now always emits `--app-cta-gradient` (a `linear-gradient(135deg, …)`
  when an end colour is set, else the solid CTA colour) and, when set, `--app-surface-color`
  and `--app-logo-bg`.
- `themeFields` (Zod) validate the three hex colours + the boolean toggle.

### Respondent surface (`components/app/questionnaire/chat/**`)

- `BrandThemeProvider` renders a **surface header band** carrying the logo (left-aligned, like
  a site header) when a surface colour is set; otherwise the centred logo as before. The logo
  paints on its resolved backdrop (a padded chip) via `--app-logo-bg`.
- `QuestionnaireChat` send button uses `background: var(--app-cta-gradient, …)` with a soft
  brand-tinted glow — a brand pill when a gradient is configured, the platform primary otherwise.

### Admin (`components/admin/demo-clients/**`)

- `DemoClientForm` "Invitation branding" fieldset is now **"Brand theming"** (it themes the
  session too), with a "Session chrome" sub-block: surface colour, CTA gradient end, and the
  "Apply a colour behind the logo" toggle (+ optional colour). Each has a `<FieldHelp>`.
- `DemoClientThemePreview` adds surface / gradient-end swatches and a **miniature of the session
  chrome** (band + logo backdrop + gradient send button) so the admin recognises the brand
  before opening "Preview as respondent".

### Email + seed

- `questionnaire-invitation.tsx` paints the solid logo backdrop behind the email logo.
- `025-demo-content.ts` (Northwind demo) sets the chrome fields so themed paths are exercised.
- The live `test-demo` (Merlin5) row was populated: `surfaceColor`/`logoBackgroundColor`
  `#280039`, `ctaColorEnd #FF03DF`, `logoBackgroundEnabled true` (existing F3.4 fields kept).

## Tests

- **Unit (theming):** `theming/theme.test.ts` extended — chrome passthrough, the four logo-
  backdrop resolution cases (off / explicit / surface-fallback / neither), and the new CSS vars
  (solid vs gradient `--app-cta-gradient`, conditional `--app-surface-color` / `--app-logo-bg`).
- Existing theme/preview/chat/seam tests updated for the wider resolved shape and select.

## Not in scope

Per-questionnaire (vs per-client) theming. Theme application to admin orchestration chat.
Font / typography theming. A colour-picker widget (hex inputs for now).

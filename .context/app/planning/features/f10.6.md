# F10.6 — Include questionnaire data in the report

> Committable tracker for **F10.6**. Lets any report mode — including the woven **narrative** — carry the
> respondent's own questionnaire data alongside the AI content: a **questions-and-answers** recap and/or
> the captured **data-slot** values. Shown on both the completion screen (card + A4 preview) and the
> downloadable PDF, driven by one config.

## Intent

- The `narrative` report was woven-only by hard rule (`SessionExportModel.narrativeOnly`): there was no
  way to also show the respondent their answers beneath it.
- The `rawIncludes` config (`questionsAsPresented` / `dataSlots`) existed and had an editor, but was
  **dead** — never consumed by any renderer, and data slots were never rendered in the PDF at all.
- An admin should be able to append the respondent's own data to the report — for narrative and the
  other modes — without touching code, and have the on-screen and PDF artifacts match.

## What shipped

1. **`rawIncludes` wired for real, all modes.** The Content-tab toggles now take effect. The editor
   (`respondent-report-editor.tsx`) shows them for **every** mode including `narrative` (previously a
   "no separate raw section" dead-end), reframed as **Include questionnaire data**.
2. **`narrativeOnly` retired.** `SessionExportModel` now carries `narrative` (title only),
   `includeQuestions`, `includeDataSlots`, and a `dataSlots` group array. `buildSessionExportModel`
   defaults: `includeQuestions: true`, `includeDataSlots: false`, `narrative: false`.
3. **Data-slot loading.** `loadSessionExport` loads `AppDataSlot` + `AppDataSlotFill` and builds
   theme-grouped `ExportDataSlotGroup[]` (value = fill paraphrase, else null → "Not captured").
4. **PDF rendering** (`session-pdf-document.tsx`) — honours `includeQuestions` for the answer listing +
   answered-count, adds a **"Captured information"** section when `includeDataSlots`, and titles the AI
   section from `narrative`.
5. **On-screen appendix** (`ReportDataAppendix` in `session-complete.tsx`) — a "Captured information"
   and/or "Your responses" block beneath the report, on the card and the A4 preview, sourced from the
   panel the respondent saw (`captured`) and gated by the report view's new `includeData` flags.
6. **Agent instructions** (`report/generate.ts`) — `APPENDED_DATA_RULES` is injected when either toggle
   is on, telling the writer the respondent can already see their answers, so analyse/synthesise rather
   than restate.

## Routing rules

- **Respondent PDF** honours `rawIncludes`, EXCEPT: when an AI report is expected but not yet embedded
  (still generating), `includeQuestions` is forced on so the download is never an empty document.
- **Admin PDF** always sets `includeQuestions: true` (full answer audit) and follows the config for the
  data-slot appendix.

## Decisions

- **PDF + on-screen** (not PDF-only): the appended data renders in both surfaces so the respondent sees
  everything without downloading.
- **All modes wired** (not narrative-only): fixing the dead `rawIncludes` for `raw` / `raw_plus_insights`
  too, so every mode behaves as its UI claims.
- **Narrative defaults to woven-only.** Selecting `narrative` in the editor resets both include toggles
  OFF (appending is opt-in); switching away restores `questionsAsPresented`. Handled in the editor's
  `changeMode` (not a config-layer default, which can't be mode-aware — `raw` needs the answers on).
- **Report grounds on the full structured audience.** `buildAnswerTranscript` renders every set
  `AudienceShape` field (role, expertise, duration, locale, sensitivity, notes) as labelled lines via
  `describeAudience`, replacing the previous one-line `summariseAudience` condensation.
- **Data-slot mode divergence** — `buildAnswerPanelView` empties `sections` in data-slot mode, so the
  on-screen Q&A recap only appears for question-mode versions (matching what the respondent saw); the
  PDF still lists the underlying questions when `includeQuestions` is on.
- No schema/migration: `rawIncludes` already existed; data slots load through existing relations.

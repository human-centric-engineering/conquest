# F9.9 — Live "watch it think" reasoning stream

> Committable tracker for **F9.9**. A respondent-facing demo surface: the per-turn orchestrator's
> reasoning — answers captured (provenance + confidence), contradictions spotted, refinements, and
> **why the next question was chosen** — streamed live beside the chat. The clearest "agent, not a
> form" signal, at **no extra LLM cost** (derived from work the turn already did).

## Intent

The orchestrator already does the work that distinguishes ConQuest from a chatbot, but the
respondent saw only a "thinking" dot then a reply. This surfaces that reasoning as a brand-themed
feed — the demo's marquee differentiator moment.

## Decisions (confirmed with the user)

- **Placement and persistence are per-questionnaire admin settings** (Settings tab) with sensible defaults — not hardcoded. See [[demo-ux-prefer-admin-config]].
- **Placement options for v1:** `overlay` (default — live animated feed, then collapses onto the turn) and `inline` (quiet collapsible disclosure). A dedicated side rail was deferred (cramps the laptop demo layout).
- **Persistence default on** — replays on resume/scroll-back; off = live-only.
- **Respondent-safe by construction** — the trace excludes the abuse/seriousness verdict and the sensitivity disclosure summary entirely; abuse-abandoned turns produce no trace.
- **Post-hoc burst, client-paced reveal** — the turn computes the whole `TurnResult` before the reply streams, so the route emits one `reasoning` frame and the client reveals the steps staggered (consistent with the existing client-side typewriter philosophy); no emitter threaded through the pure orchestrator.

## What shipped

**Pure core**

- `lib/app/questionnaire/reasoning/` — `ReasoningStep` types + `buildReasoningTrace(result, opts)`, mapping a `TurnResult` to a respondent-safe `ReasoningStep[]` in pipeline order. Unit-tested (both modes, PII exclusion, abuse-abandon, opening turn, per-strategy selection phrasing).
- `TurnResult.selectionRationale` / `selectionStrategy` — the selector's rationale lifted onto the result (otherwise dropped) in `orchestrator.ts` + `data-slot-orchestrator.ts` (the latter phrases deterministic data-slot moves).

**Wire + persistence**

- New `{ type: 'reasoning', steps }` SSE frame emitted by the `/messages` route (gated by flag + config); `SessionStreamEvent` + `parseSessionEvent` narrow it (drop malformed steps).
- Additive migration `20260615094314_reasoning_stream`: 3 `AppQuestionnaireConfig` columns (`reasoningStreamEnabled` / `reasoningStreamPlacement` / `reasoningStreamPersist`) + `AppQuestionnaireTurn.reasoning` (JSON). Generated `--create-only`, phantom pgvector DDL stripped. Written in `recordTurn`/`persistTurn` (gated by `persist`); replayed via `loadTranscript`.

**Flag / config / UI**

- Sub-flag `APP_QUESTIONNAIRES_REASONING_STREAM_ENABLED` (DB row, seed `039`, off by default) + `isReasoningStreamEnabled()` (master + live-sessions + sub-flag).
- Config wired through `updateConfigSchema`, `QuestionnaireConfigShape` + defaults, `REASONING_PLACEMENTS` enum, the `detail.ts` read view, and a new "Reasoning stream" group in `config-editor.tsx` (Settings tab) with `<FieldHelp>` on each control.
- `reasoning-trace.tsx` (live + collapsed variants, brand-themed via `--app-accent-color`, `cq-rise` + reduced-motion, signal-strength confidence pips); threaded through `questionnaire-chat.tsx`, `session-workspace.tsx`, both respondent pages, and `AnonymousSessionBoot`. `use-questionnaire-session-stream` exposes `streamingReasoning`.

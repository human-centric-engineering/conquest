# F9.5 — Seriousness / abuse gate

> Committable tracker for **F9.5**. Per answered turn, judge whether a respondent's answer is a
> genuine attempt; disregard non-genuine ones (preposterous / abusive / off-topic — e.g.
> "543 years" of tenure), strike the session, escalate a warning, and abandon at a configurable
> threshold — while tolerating colloquial / lazy answers. Mirrors contradiction detection (F4.3).

## Intent

A respondent could give an absurd answer ("543 years") and the agent accepted it credulously,
polluting results. F9.5 adds a gate that politely sets aside non-genuine answers, escalates, and
ends the session after `abuseThreshold` strikes, recording an analytics-visible abandonment.

## Decisions (confirmed with the user)

- **Two-stage, cost-efficient.** The answer extractor (the pass already reading the answer) flags
  `suspectedNonGenuine` (no extra call); a dedicated LLM judge runs ONLY on suspicion.
- **Configurable threshold, default 4.** Strikes 1–3 escalate warnings, the 4th abandons. `0` = off.
- **Disregard + re-ask.** A flagged answer is never persisted; the same question is re-asked via a
  side-band notice.

## What shipped

- **Schema:** `AppQuestionnaireConfig.abuseThreshold Int @default(4)`,
  `AppQuestionnaireSession.abuseStrikes Int @default(0)` (migration
  `…_app_seriousness_abuse_gate`, create-only + phantom-DDL stripped).
- **Pure core:** `lib/app/questionnaire/seriousness/**` (verdict schema + judge prompt + escalation
  logic); orchestrator branch in `orchestrator.ts` (judge **every** answered turn while the gate is
  on → disregard/warn/abandon), `TurnState.abuseStrikes`, `TurnFlags.seriousnessGate`,
  `TurnResult.abuse`, `ExtractOutcome.suspectedNonGenuine`, `assessSeriousness` invoker.
- **Suspicion hint:** the extraction Zod schema + prompt still carry `suspectedNonGenuine`, but it is
  a diagnostic only — the final design runs the judge on every answered turn rather than gating it on
  the extractor's (unreliable) suspicion flag.
- **Judge:** a direct structured LLM call in `turn-invokers.ts` (reuses the extractor
  binding — no new seeded agent/capability).
- **Route:** `messages/route.ts` resolves the gate flag, persists strikes, and abandons at the
  threshold (`abandonSession` reason `abuse_threshold_exceeded` + metadata); `persistAbuseStrikes`
  seam in `sessions.ts`; `abuseStrikes` loaded by `turn-context.ts`; `abuseThreshold` flows through
  `detail.ts` (`CONFIG_SELECT`/`toConfigView`) + the config Zod + PATCH spread.
- **Gating:** platform flag `APP_QUESTIONNAIRES_SERIOUSNESS_GATE_ENABLED`
  (`isSeriousnessGateEnabled()`, seed `029-seriousness-gate-flag.ts`, default off).
- **UI:** config-editor "Abuse threshold" field; `SeriousnessNotice` wired to
  `warning.code === 'seriousness'`. Abandonment reuses the existing `abandoned` → terminal-status
  lock.

## Addendum (2026-06-16) — deterministic abuse floor

**Why:** With sensitivity awareness on (F9.6), abuse that followed a genuine disclosure repeatedly
went unstruck. Tracing the live turns showed two distinct, _probabilistic_ failure modes, neither
fixable by prompt tweaks: (a) the dedicated sensitivity detector intermittently read later pure abuse
("oh just fuck off") as the discloser's _distress_ and flagged it sensitive, which skipped the gate;
(b) even when the gate ran, the LLM **judge** — fed the disclosure in its recent-conversation context
— intermittently returned `serious: true` for plain dismissals. Same message, same code, different
result across runs (confirmed: one session struck "fuck off", another kept it).

**Fix:** added a **deterministic abuse floor** (`seriousness/abuse-net.ts`, `keywordAbuseFloor`),
mirroring the keyword _sensitivity_ net. A short message dominated by a directed-dismissal phrase is
struck with **no LLM call**. It is tight (only directed dismissals, not bare insults; only short
messages, so reported phrases in a sentence defer to the judge), fires even when an LLM flagged the
turn sensitive (an over-eager detector can't shield plain abuse), and is suppressed only by the
deterministic HARM floor (abuse + a real disclosure stays protected). The gate block in both
orchestrators was restructured: floor first → else judge (only when not deterministically abusive and
not a detected disclosure). The judge + scoped prompts remain for nuanced/longer abuse.

**Verified live** against the dev server: disclosure turn skips the gate + signposts; the three
following abuse turns ("oh just fuck off", "screw you", "go fuck yourself") are each struck
deterministically (`abuseStrikes` reaches 3, escalating notices), `sensitivityLevel` stays `high`.
Floor strikes record an `app_assess_seriousness` tool call with no `latencyMs` (no LLM call). Tests:
`seriousness/abuse-net.test.ts` + orchestrator cases (strikes-without-judge, overrides-LLM-detector,
harm-floor-suppresses-abuse-floor).

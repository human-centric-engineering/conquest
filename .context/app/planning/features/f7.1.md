# F7.1 — Chat surface

> Committable tracker for **F7.1**, the opening feature of P7. The respondent-facing surface
> where someone completes a questionnaire through a streaming conversation: a bespoke SSE chat
> client over the F6.1 `/messages` contract, voice input via the F6.2 transcribe endpoint, the
> F3.4 demo-client branding applied, and a Playwright demo-flow e2e. Pure consumer UI — it adds
> **no new API endpoints**. Gated by `APP_QUESTIONNAIRES_ENABLED` + `APP_QUESTIONNAIRES_LIVE_SESSIONS_ENABLED`.

## Intent

P6 made the turn loop live (server-side). F7.1 is the thing a prospect actually sees: render that
streamed conversation as a real respondent experience across all three F6.1 access modes
(invitation-bound, logged-in anonymous-mode, and the no-login public surface), wire voice and
branding, and protect the sales-critical happy path with an end-to-end test.

## Decisions (confirmed with the user)

- **Bespoke chat client, not the admin `ChatInterface`.** The admin component is hardwired to the
  orchestration `agentSlug` stream; the questionnaire `/messages` contract differs (session-id URL,
  `X-Session-Token` header, a smaller `start/content/warning/done` event subset, 402/409 pre-stream
  statuses). A new `QuestionnaireChat` consumes that contract but **reuses** the platform sub-pieces:
  `MicButton`, `useTypingAnimation`, `ThinkingIndicator`, `parseSseBlock`, and the
  `fetch` + `ReadableStream.getReader()` reader pattern.
- **Both access modes from the start.** One hook (`useQuestionnaireSessionStream`) drives all modes;
  the only runtime difference is whether it attaches `X-Session-Token`. The authed surface lives under
  `(protected)` (cookie auth, ownership verified in-page since `/questionnaires` isn't in the proxy's
  protected list); the no-login surface lives under `(public)/q/[versionId]` and mints its session
  **client-side** so the signed token never lands in server-rendered HTML.
- **Cost cap is terminal for the session.** A pre-stream 402 (`COST_CAP_REACHED`) maps to a dedicated
  `cost_capped` state that disables the composer — not a retryable error — to avoid a confusing
  402→409 flip (the next turn would 409 the now-paused session). 409 → `not_active`, anonymous 401 →
  `expired` (reload to restart), 429 → transient error (retry allowed).
- **No transcript replay.** Resume restores answered slots only (no message-history endpoint), so a
  resumed session opens with a synthetic "welcome back" greeting rather than faking prior bubbles. A
  fresh session opens with the branded `welcomeCopy` + a "send a message to begin" nudge (the loop
  only responds to respondent input).
- **Attachments deferred.** `/messages` accepts `{ message }` only; attachment input is blocked on a
  future F4.2 extension and is omitted from F7.1.
- **`MicButton` gains `extraHeaders`.** A backward-compatible optional prop so the no-login path can
  send `X-Session-Token` (the platform `MicButton` was cookie-only). The one platform-surface change
  in F7.1 → its only CHANGELOG entry.
- **Branding reuses F3.4.** The theming module already exists; F7.1 consumes `resolveTheme` +
  `themeToCssVariables` (colours as `--app-cta-color` / `--app-accent-color`, escaped logo via
  `--app-logo-url`) and the resolved `welcomeCopy`. `// DEMO-ONLY` — a fork strips the theme seam.

## PR breakdown

- **PR1 — core chat + both routes.** Hook, `QuestionnaireChat`, error panel, anonymous boot,
  greeting, `parse-session-event`, `session-bootstrap`, the `API.APP.QUESTIONNAIRE_SESSIONS`
  endpoints block, and the protected `start` + `[sessionId]` pages + public `/q/[versionId]` page.
- **PR2 — voice.** `MicButton.extraHeaders`; wired into `QuestionnaireChat` (`agentId = sessionId`,
  token header for anon); gated on `isVoiceInputEnabled()` server-side.
- **PR3 — branding.** `BrandThemeProvider` + the `theme.ts` Prisma seam (session/version →
  questionnaire → demo client); applied on both pages with the resolved `welcomeCopy`.
- **PR4 — Playwright e2e.** `@playwright/test` dev-dep, `playwright.config.ts`, `test:e2e` scripts,
  the demo happy-path + flag-gate specs, and `tests/e2e/README.md`.

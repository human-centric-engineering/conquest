# F14.10 — Remove the questionnaire feature-flag layer

> Committable tracker. The questionnaire product accreted a master flag plus ~34 sub-flags
> (`APP_QUESTIONNAIRES_*_ENABLED`, `APP_REPORT_FORMATTER_ENABLED`) as `feature_flag` rows,
> all seeded disabled. This became friction — every feature off by default, toggled per
> environment. They are now removed: **every questionnaire feature is permanently on**, and
> the only runtime toggle left in the whole system is site-wide maintenance mode.

## Intent

Delete the questionnaire flag layer entirely (not shim it), default all features on in every
environment with no dependence on DB state, and keep the generic Sunrise flag infrastructure and
`MAINTENANCE_MODE` untouched. Reintroduce a flag only when explicitly asked, at that time.

## What shipped

- **Read-seam flip first**: neutralised the resolver hub and the workspace/report bypasses so
  every feature reads as on regardless of `feature_flag` rows — the cross-environment safety
  net (no prod backfill needed).
- **Full unwind**: removed `with*Enabled` route wrappers, `ensure*Enabled` inline gates,
  `is*Enabled` toggles, and `flags.X` page gates across ~200 files; the workspace nav no longer
  flag-filters tabs. Per-feature behaviour is now governed solely by each version's own config
  toggles (the author opt-ins that were always the second half of the old gate).
- **Deleted**: `lib/app/questionnaire/feature-flag.ts`, all `APP_*_FLAG` constants in
  `constants.ts`, `resolveQuestionnaireWorkspaceFlags` + `QuestionnaireWorkspaceFlags`, the
  `report-flag` gate, and the 35 `prisma/seeds/app-questionnaire/*-flag.ts` seed units.
- **DB cleanup**: migration `20260716160000_remove_app_questionnaire_feature_flags` deletes the
  orphaned `APP_*` rows on `migrate deploy` (fires on the live Vercel env where seeds don't run);
  prefix-scoped so it cannot touch `MAINTENANCE_MODE`; idempotent.
- **Tests + docs** updated: flag-disabled cases and flag mocks removed, tests of deleted seeds/
  resolvers deleted; the flag docs rewritten to a historical note.

## Deployment note

At the next deploy, any questionnaire sub-feature that was disabled in an environment turns on
— including paid per-turn LLM work, Brave web search, and the public no-login respondent surface.
This is intended (live demos). Cost-cap enforcement also becomes active, which pauses sessions
that hit their configured USD budget.

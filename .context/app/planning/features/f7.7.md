# F7.7 — Confidence nuance

> Committable tracker for **F7.7**. Capture confidence was landing at ~0.8 ("Fairly sure") for
> almost everything because the extraction prompt's rubric defaulted to a coarse three-band
> CLEAR(~0.8)/PARTIAL(~0.5)/UNCLEAR(≤0.4) scheme and explicitly forbade marking clear answers down.
> This refines confidence end-to-end so it spans ~0.3–1.0, makes the nuance visible, and steers the
> conversation to deepen shaky answers.

## Intent

Make confidence **mean something** across its range. Score it by **how** an answer was given —
directness × elaboration × certainty — so a directly-stated, example-backed answer reads high and a
terse or tangentially-inferred one reads low. Then put that nuance to work: show it to the respondent
(per-answer % + an average-confidence header), and bias next-question selection to **probe deeper**
where confidence is low while moving on where it's high.

## Decisions (confirmed with the user)

- **Per-answer confidence shows the raw % to respondents** (e.g. "Fairly sure · 62%"), not just a
  band word — the nuanced 30–100% range is made visible. (The `ConfidenceScore` chip already rendered
  the number; this confirms it as the intended respondent-facing behaviour, relaxing the earlier
  "felt, not totted up" stance.)
- **Header average uses all filled slots** — an honest mean (a tangential, low-confidence fill drags
  it down by design), shown as a number alongside completion.
- **Probing depth is admin-configurable** — surfaced via the existing `maxDataSlotAttempts` on the
  Settings tab (default unchanged); a higher value probes shaky answers harder at the cost of a longer
  conversation.

## What shipped

### Confidence rubric rewrite (`lib/app/questionnaire/extraction/extraction-prompt.ts`)

- The question-answer band, the data-slot-fill band, and the mapped-answer band moved from a
  three-band ~0.8-default scheme to **four numeric anchors over 0.3–1.0** keyed on directness ×
  elaboration × certainty: direct+backed 0.9–1.0; clear-direct 0.75–0.85; terse/vague 0.45–0.6;
  tangential inference 0.3–0.45. The anti-nuance clauses ("never mark a clear answer down for being
  brief… / score the substance") were removed.
- **Closed-question carve-out:** a complete, unambiguous closed answer (choice/likert/numeric/date/
  boolean) stays high (0.85–0.95) even when terse — only an uncertain _mapping_ drops it.
- The `rationale` now opens with the basis for a non-obvious score ("Stated directly with an
  example…", "Inferred tangentially from their comment about…").
- The park instruction's low-confidence floor aligned to 0.3–0.4.

### Eval recalibration (`extraction/eval/`)

- `score.ts`: kept three coarse bands (an LLM float isn't trustworthy at finer resolution) but moved
  `PARTIAL_BAND_MIN` 0.4 → **0.45** to sit on the rubric's terse/tangential seam; documented the
  band↔rubric mapping.
- `golden-set.ts`: rebalanced — fixture #4 (strong sentiment) now expects `clear`; added a
  terse-but-complete closed answer (clear), a terse qualitative answer (partial), and a tangential
  inference (unclear, flagged `knownGap`) so the new spread is measured.

### Thresholds

- `DATA_SLOT_FILLED_THRESHOLD` kept at **0.5** — a `direct` answer is covered by provenance
  regardless of its number, so only weak inferred/synthesised fills are gated (the intended deepen
  trigger). Completion stays question-coverage-gated, never confidence-gated.
- `selection/types.ts`: `LOW_CONFIDENCE_THRESHOLD` 0.5 → **0.6** so the weighted scorer treats the
  terse band (~0.45–0.6) as shaky ground; `LOW_CONFIDENCE_MULT` unchanged (1.5).

### Deepen-on-low-confidence selection

- `selection/strategies/adaptive.ts` + `selection/types.ts`: each candidate now carries an optional
  `sectionLowConfidence` flag (its section already holds a ≤0.6 answer), computed from `ctx.answered`.
- `_lib/adaptive-deps.ts` (`buildSelectorPrompt`): renders a "We're unsure here — probe to deepen it"
  sub-line and adds a deepen nudge to the task framing.
- Seed `005-selection-agent` (`SELECTOR_INSTRUCTIONS`): a "DEEPEN SHAKY GROUND" clause. **Requires a
  re-seed (fresh DB) or admin edit to take effect** — re-seeding only re-asserts `isSystem`.
- `_lib/question-stream.ts`: the re-ask/deepen turn-guidance now names _why_ it's circling back
  ("Earlier you mentioned …, and I want to make sure I follow…"), grounded in `currentUnderstanding`.

### Display layer

- `panel/confidence.ts`: four scored bands — high ≥0.85 "Confident", moderate ≥0.65 "Fairly sure",
  **tentative ≥0.45 "Tentative"**, low <0.45 "Unsure" (+ unscored "Captured"), with an
  emerald→amber→orange→red tint ramp. Deliberately decoupled from the admin eval chips.
- `_lib/answer-panel.ts` + `panel/types.ts`: `AnswerPanelView.averageConfidence` — mean over the
  data-slot fills (data-slot mode) or answered question slots (question mode); the panel header shows
  "… · avg confidence NN%".
- `panel/answer-slot-panel.tsx`: header pairs completion with the average; the now-accurate comments
  note respondents see the score.

### Config

- `config-editor.tsx`: `maxDataSlotAttempts` (already on the Settings tab) `<FieldHelp>` expanded to
  explain it's the ceiling on how hard a shaky/low-confidence answer is deepened before parking.

## Tests

- **Unit:** confidence band boundaries (4 bands + Tentative label/orange tint); eval `classifyBand`
  0.45 cut; extraction-prompt park-band assertion; weighted strategy 0.55→trips raised threshold;
  adaptive `sectionLowConfidence` flagging; `buildSelectorPrompt` deepen nudge; reasoning-trace band
  words (added tentative); `ConfidenceScore` tentative band; reasoning-trace pip aria-label.
- **Integration:** `answer-panel.test.ts` — `averageConfidence` in question mode (mean over scored
  answers, omitted when none scored) and data-slot mode (mean over fills).
- `npm run validate` green; full questionnaire suites pass.

## Not in scope

- Adding average confidence to the top lifecycle bar / `SessionStatusView` — deliberately
  consolidated in the answer-panel header (the data-slot surface the request named) to avoid two
  independently-computed averages that could disagree.
- Refining the admin evaluation chips to match the finer respondent bands (the chips stay two-cut).

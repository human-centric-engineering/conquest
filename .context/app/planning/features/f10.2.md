# F10.2 — Respondent Report mode 3 (structured narrative)

> Committable tracker for **F10.2**. The third and final report mode from the original brief, deferred
> to v2 by [F10.1](./f10.1.md): a **structured narrative** that weaves the respondent's answers into
> one flowing, analysed report (analyses, insights, suggestions, advice) — branded, downloadable as
> PDF, optionally grounded in the per-client KB — rather than F10.1 mode 2's separate raw + insights
> sections.

## Intent

After completing a questionnaire, a respondent in narrative mode receives a single woven report: their
answers are integrated into continuous analytical prose, not listed as a separate raw section. Admins
select it per version from the existing **Respondent report** workspace tab, using the same generation
config (instructions / structure / background context), config-crafting chat, and per-client KB the AI
insights mode already had. Gated by the same platform flag
`APP_QUESTIONNAIRES_RESPONDENT_REPORT_ENABLED`.

## Agreed scope

Add `mode: 'narrative'` as a third value alongside `raw` and `raw_plus_insights`. Two product
decisions (confirmed with the user):

1. **Reuse the structured content shape** `RespondentReportContent` (`{ summary, sections[], actions[] }`)
   with a narrative-tuned prompt — sections become woven chapters that integrate the answers. No new
   content type, **no migration**, no new agent.
2. **Woven report only** — the narrative is the whole deliverable; the respondent's PDF + on-screen view
   show no separate raw answer / data-slot list. The admin audit PDF still shows everything.

## What shipped (by phase, one commit each)

1. **Backend** — `'narrative'` added to `RESPONDENT_REPORT_MODES` + the shared `isAiRespondentReportMode`
   predicate (so the two AI modes never drift). `buildReportMessages` (`report/generate.ts`)
   mode-branches the prompt (woven framing + woven-chapter sections). `enqueue` / `view` / poller all
   gate on `isAiRespondentReportMode`, so narrative enqueues, generates async, and surfaces content.
   Schema comment only (the `mode` column is a plain `String`).
2. **Respondent delivery (woven-only)** — `SessionExportModel.narrativeOnly` threaded through
   `buildSessionExportModel` / `buildSessionExportPdfModel`. `SessionPdfDocument` renders the report
   under "Your personalised report" and omits the raw section/slot listing + answered-count when
   `narrativeOnly`. The respondent `export.pdf` route sets it for a ready narrative report; the admin
   route never does (full audit + report). `SessionComplete` renders narrative on-screen via the
   existing `ReportInsights` (the completion screen lists no raw answers, so it is woven-only there too).
3. **Admin config UI + tracker** — the editor's mode selector gains **Narrative report**; the
   generation panel / config-crafting chat / KB panel are gated on `isAiRespondentReportMode` (enabled
   for narrative); the **Content** tab hides the raw-content toggles in narrative mode (replaced by a
   notice). FieldHelp copy updated. This tracker + doc.

## Key decisions

- **Reuse, don't fork** — same content shape, agent (045), config-crafting assistant (046), worker,
  and KB isolation as mode 2. The only narrative-specific code is the prompt branch and the woven-only
  render path; everything else is "extend `mode === 'raw_plus_insights'` to `isAiRespondentReportMode`".
- **No migration** — the `mode` column is a plain `String` validated at the app layer; `content` JSON
  is reused.
- **Woven-only is respondent-scoped** — `narrativeOnly` is set by the respondent PDF route, not the
  model builder, so the admin audit PDF keeps the full answer record alongside the report.

## Tests

Narrative generation prompt branch; enqueue queues narrative; view exposes insights for narrative;
settings + config-schema accept narrative; poller polls narrative; narrative PDF render + flag
passthrough; respondent route `narrativeOnly` wiring (narrative / mode-2 / not-ready); on-screen
narrative render + preparing state; editor narrative option, generation enabled, raw toggles hidden +
KB panel for narrative. `npm run validate` clean.

## Not in scope / follow-ups

- A visually distinct narrative PDF typography (currently reuses the insights styling under a new

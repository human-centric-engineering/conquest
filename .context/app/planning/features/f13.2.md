# F13.2 — Round Learning Mode

> Committable tracker for **F13.2**, the second cohort-round capability. The interviewer is given
> generalised, anonymised themes from EARLIER respondents in the same round and uses them subtly — to
> colour phrasing AND (under the adaptive strategy) to probe divergent topics harder. Off by default;
> introduces bias by design. Built across phases 1 (shared foundation), 4 (backend) and 5 (admin UI).

## Intent

A skilled human consultant interviewing people one after another can't help noticing patterns, and
gently brings them into later conversations ("a few people mentioned X — how do you feel about
that?"). That produces richer depth — at the cost of biasing later answers. Learning Mode gives the
agent that same ability, made safe with anonymisation + a k-anonymity gate and made honest with an
explicit bias warning.

## Decisions

- **Round-scoped corpus** — only completed, non-preview sessions in the SAME round + version. The
  current respondent is excluded structurally (their session is still `active`, not in the corpus).
- **Cached digest, rebuilt on completion** — `AppRoundLearningDigest` holds one generalised theme +
  divergence per slot, rebuilt by ONE composer-agent call when a session submits (so the next
  respondent sees the just-finished one). Read cheaply each turn; never re-aggregated live.
- **Privacy by construction** — generalised text only (no names, no quotes, no individual
  attribution); high-sensitivity sessions excluded wholesale; k-anonymity (`minRespondents`, default
  3, floor 2) gates BOTH the round and each slot. Wholesale transactional rebuild so a shrunk corpus
  (e.g. erasure) can't leave stale rows; fail-soft never wipes on a transient LLM error.
- **Phrasing AND adaptive probing** (the richer option) — peer themes colour phrasing for every
  strategy; per-question divergence additionally nudges the `adaptive` selector to probe split topics
  (other strategies stay phrasing-only).
- **Bias is surfaced, not hidden** — always-on warning in the panel, a `Learning · biased` pill in the
  round header, and a per-session `learning_applied` audit event when peer context actually fires.

## What shipped

- **Foundation** (phase 1, with F13.1) — `AppRoundLearningDigest`, `learningEnabled` +
  `learningConfig`, `isLearningModeEnabled`, seed `051`, `resolveLearningConfig` + `minRespondents`.
- **Backend** (phase 4) — `lib/app/questionnaire/learning/digest.ts`
  (`refreshRoundLearningDigest` / `loadRoundPeerDigest` / `listRoundLearningDigest`),
  `learning/events.ts` (`recordLearningApplied`); submit-route rebuild trigger; `<peer_context>`
  prompt section; adaptive probing via `SelectionContext.peerDivergenceByKey` → the selector prompt.
- **Admin UI** (phase 5) — `RoundLearningPanel` (bias warning, toggle, `minRespondents`, theme
  preview with divergence bands, Rebuild), `POST …/learning/rebuild` (`withLearningModeEnabled`), the
  header bias pill, and the erasure story (rebuild-on-completion + manual rebuild; no `eraseUser` hook
  needed since the digest holds no individual data).

## Gates

`isLearningModeEnabled()` (master + cohorts + learning sub-flag) AND the per-round `learningEnabled`
toggle AND the k-anonymity threshold being met. Generalisation reuses the seeded composer agent.
Adaptive probing additionally requires the `adaptive` selection strategy.

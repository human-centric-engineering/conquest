# F9.6 — Sensitivity awareness / safeguarding

> Committable tracker for **F9.6**. When a respondent discloses something sensitive or contentious
> (abuse, distress, safeguarding), detect it that turn, remember it at the session level, phrase
> every later question more gently, and gently signpost support once on a serious disclosure.
> Mirrors the seriousness gate (F9.5) end-to-end.

## Intent

The interviewer only had a static, author-declared `audience.sensitivity` line — it didn't notice
what was _just said_, didn't remember it, and asked the next question with the same neutral tone.
F9.6 makes the conversation tread carefully after a disclosure, and surfaces a lightweight
safeguarding signal to admins.

## Decisions (confirmed with the user)

- **Detection folds into the answer-extraction call** (no new always-on LLM call).
- **Admin = a count in the existing analytics tab + a session event** (no full session viewer yet).
- **Respondent UX = soften tone AND signpost support** (a configurable, verbatim support message).
- Explicitly **best-effort awareness, not a guaranteed safeguarding net** (detection rides on the
  extractor's structured side-signal).

## What shipped

- **Schema:** `AppQuestionnaireConfig.{sensitivityAwareness,supportMessage,supportResourceUrl}`,
  `AppQuestionnaireSession.{sensitivityLevel,sensitivityNotes}` (migration
  `…_app_sensitivity_awareness`, create-only + phantom pgvector DDL stripped).
- **Pure core:** `lib/app/questionnaire/sensitivity/**` (severity rank, running-max, signpost,
  support-message assembly); `SENSITIVITY_SEVERITIES` + `SENSITIVITY_FLAGGED_EVENT` in `types.ts`;
  orchestrator step "1.6" in both `orchestrator.ts` and `data-slot-orchestrator.ts`;
  `TurnState.{sensitivityLevel,sensitivityNotes}`, `TurnFlags.sensitivityAwareness`,
  `ExtractOutcome.sensitivity`, `TurnResult.sensitivity`.
- **Detection:** optional `sensitivity` object on the extraction Zod schema + a conditional prompt
  block gated by `ExtractionContext.sensitivityAware`; threaded through the extractor capability
  (`redactProvenance` records severity+category, never the summary) and `turn-invokers.ts`.
- **Phraser:** `question-stream.ts` gains `sensitivityLevel`/`sensitivityNotes` inputs + a
  tread-carefully system block; the route folds the just-detected note in for the disclosure turn.
- **Persistence + route:** `persistSensitivity` + `recordSensitivityFlagged` (`sessions.ts`); memory
  loaded by `turn-context.ts`; config flows through `detail.ts` + the config Zod + PATCH spread;
  `messages/route.ts` resolves the flag, threads phraser inputs, persists note/level + writes the
  event.
- **Signpost:** a deterministic side-band `support` frame (verbatim copy + optional URL), once per
  session on the first `high`, rendered by `support-notice.tsx` + a `warning.code === 'support'`
  branch in `questionnaire-chat.tsx`.
- **Gating:** platform flag `APP_QUESTIONNAIRES_SENSITIVITY_AWARENESS_ENABLED`
  (`isSensitivityAwarenessEnabled()`, seed `032-sensitivity-awareness-flag.ts`, default off) +
  per-questionnaire `sensitivityAwareness` toggle. Config-editor "Sensitivity awareness" group.
- **Analytics:** `getSafeguardingSummary` (counts only, k-anon suppressed) +
  `GET …/analytics/safeguarding` + a small tile on the analytics tab.

## Not in scope

A dedicated per-session safeguarding viewer / transcript UI; a two-stage dedicated judge (detection
is folded into extraction by decision); configurable detection model/persona.

the Sunrise platform contract — consistent with F1.1–F9.5.

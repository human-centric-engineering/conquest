# F6.2 — Voice input

> Committable tracker for **F6.2**. Lets a respondent answer by voice: a transcribe endpoint
> turns recorded audio into text via Sunrise's audio provider (OpenAI Whisper), and the client
> sends the transcript through the existing text turn loop. Gated by `APP_QUESTIONNAIRES_ENABLED`
> plus a new `APP_QUESTIONNAIRES_VOICE_INPUT_ENABLED` dark-launch sub-flag.

## Intent

Sunrise already ships a complete, production-tested audio stack wired to Whisper — provider
resolution (`getAudioProvider`), the `transcribe()` interface, MIME/size validation, the
`audioLimiter`, and the `'transcription'` cost-log shape. F6.2 is mostly **wiring**, not new
capability: expose those primitives to respondents through an app-owned transcribe endpoint,
dark-launched behind its own sub-flag.

## Decisions (confirmed with the user)

- **Separate transcribe endpoint, not folded into `/messages`.** The indicative plan said
  "`/messages` accepts multipart audio", but that collides with P7's "consume `<MicButton>`
  verbatim" — Sunrise's `<MicButton>` expects an endpoint that returns `{ text }`, not an SSE
  turn stream. So F6.2 adds `POST /api/v1/app/questionnaire-sessions/:id/transcribe` returning
  `{ text, durationMs, language? }`. P7 points `<MicButton>` at it verbatim, the respondent can
  edit the transcript before sending, and the text `/messages` path + SSE loop stay untouched.
- **New dark-launch sub-flag + `audioLimiter`.** `APP_QUESTIONNAIRES_VOICE_INPUT_ENABLED` (seed
  022, off by default) gates the route — every call spends per-minute Whisper cost, so an operator
  opts in deliberately. `isVoiceInputEnabled` requires master + **live-sessions** + voice: voice
  depends on the live surface (a transcript is only useful once it can be sent through the live
  `/messages` loop), so it's an opt-in on top of live-sessions, not an independent surface — turning
  live-sessions off also closes the paid audio path. (This composition was tightened during the
  `/code-review` gate; the first cut gated on master + voice only, which left a reachable-but-dead
  paid path when live-sessions was off.) The route applies Sunrise's `audioLimiter` (10/min) as a
  per-flow sub-cap, keyed `audio:qn:<rateKey>` so respondent keys don't collide with the
  admin/embed callers on the shared limiter.
- **Reuse Sunrise primitives; don't fork-edit the platform.** The route reuses
  `getAudioProvider`/`transcribe`, `enforceContentLengthCap`, the size cap / MIME allowlist
  **constants**, `logCost`, and `resolveTurnAccess` (all three respondent-access modes). The one
  app-side addition is `validateAudioUpload` — the platform's `validateTranscribeUpload` minus its
  mandatory `agentId` (the session supplies agent context), reusing the platform constants so the
  accepted formats/limit can't drift.
- **No audio persistence.** Audit invariant (mirrors the admin transcribe route): the handler
  never writes audio bytes or the transcript — the only happy-path DB write is the
  fire-and-forget `logCost({ operation: 'transcription' })` (no `agentId`; `sessionId` in
  metadata).

## Key files

- Route: `app/api/v1/app/questionnaire-sessions/[id]/transcribe/route.ts`.
- Validator: `app/api/v1/app/questionnaire-sessions/_lib/audio-upload.ts` (`validateAudioUpload`).
- Flag: `lib/app/questionnaire/constants.ts` (`APP_QUESTIONNAIRES_VOICE_INPUT_FLAG`),
  `lib/app/questionnaire/feature-flag.ts` (`isVoiceInputEnabled` / `withVoiceInputEnabled`).
- Seed: `prisma/seeds/app-questionnaire/022-voice-input-flag.ts` (disabled by default).
- Reused (Sunrise): `lib/orchestration/llm/provider-manager.ts` (`getAudioProvider`),
  `lib/validations/transcribe.ts` (constants + `enforceContentLengthCap`),
  `lib/orchestration/llm/cost-tracker.ts` (`logCost`), `lib/security/rate-limit.ts`
  (`audioLimiter`), `_lib/turn-access.ts` (`resolveTurnAccess`).

## Deferred / out of scope

- **UI** — `useVoiceRecording` + `<MicButton>` wiring is **P7** (F7.1). This endpoint is the
  contract P7 consumes.
- **Cost-cap firing** on voice spend — F6.3.
- **Admin-side anonymous redaction** of transcripts — a later phase.

## Tests

`tests/integration/api/v1/app/questionnaire-sessions/transcribe-route.test.ts` (27 tests): gate
order (flag → session → access → status → audio sub-cap → size guard → multipart → validation →
provider), all three access modes, the full audio-validation matrix (missing/empty/oversize/bad
MIME/bad language/non-multipart), provider routing (503/502 + no message leak), the happy path
(response shape, `transcription` cost-log with `sessionId` metadata and no `agentId`,
language/filename/MIME forwarding), the pre-parse size guard, and the no-audio-persistence audit
invariant. No CHANGELOG entry (app surface, not the Sunrise platform contract).

# F9.8 — Generative authoring (compose from brief)

> Committable tracker for **F9.8**. A new input path: an admin describes a goal in
> plain English and an agent composes the whole questionnaire — sections, questions,
> types — streaming it into existence, then refines it conversationally before it
> lands in the Structure editor. The sibling of document extraction, reusing its
> output contract, persistence graph, and streaming surface — **no new Prisma models,
> no migration.**

## Intent

Every prior path into the system required a **document**. The extractor parses an
upload; there was no "type a goal → AI composes from scratch" capability. This
collapses time-to-wow from minutes to seconds and is the most self-evidently
"agentic, not a form-builder" moment we can stage: the prospect says "make me an
onboarding survey for B2B SaaS churn risk" and watches sections + questions stream
in, then tweaks it by chatting.

## Decisions (confirmed with the user)

- **Conversational refine is in v1** — initial one-shot streamed generation, then an iterative chat loop ("make it shorter", "add a section on pricing") before saving.
- **No auto-chain to data-slots** — compose generates structure only; the admin runs data-slot generation in the editor as today. A true sibling, input-path only.
- **Combined dropdown entry** — the list page's single upload button becomes a "New questionnaire" menu: "Upload document" + "Describe your goal".

## What shipped

**Backend**

- Two capabilities (`app_compose_questionnaire`, `app_refine_questionnaire_structure`) extending `BaseCapability`, both `processesPii = true`, reusing the extractor's `extractionSchema` output (with an **empty** change log). `capabilities/{compose-questionnaire,refine-questionnaire-structure}.ts`, barrel + `lib/app/capabilities.ts` registration.
- Two-phase streaming orchestrator `ingestion/stream-compose.ts` (outline → per-section fan-out at concurrency 4 → cross-section key de-dup), with `compose-prompt.ts`, `compose-schema.ts` (derived from `extractionSchema`), `compose-events.ts`.
- Persistence helpers `briefSource` (synthesizes the source-doc row from the brief) + `replaceVersionStructure` (transactional clear-and-rewrite for refine) in `_lib/persist.ts`; `_lib/compose-pipeline.ts` (load composer agent, dispatch, `loadRefinableStructure` draft/no-session guard); `_lib/compose-input.ts` (Zod bodies).
- Three routes: `compose` (non-streaming, API-first), `compose/stream` (SSE, persists at terminal, `done` carries new ids), `:id/versions/:vid/compose/refine`. All gated by `withGenerativeAuthoringEnabled`, per-admin `composeLimiter` (20/min). Endpoints added to the API client map.

**Flag / seeds / agent**

- Sub-flag `APP_QUESTIONNAIRES_GENERATIVE_AUTHORING_ENABLED` (DB row, seed `035`, off by default) + `isGenerativeAuthoringEnabled` / `ensureGenerativeAuthoringEnabled` / `withGenerativeAuthoringEnabled`.
- Dedicated composer agent (seed `036`, provider-agnostic, reasoning tier) + two capability seeds (`037`, `038`).

**Admin UI**

- `NewQuestionnaireMenu` replaces the upload button (drives the existing upload dialog in controlled mode + routes to the studio).
- Compose Studio (`app/admin/questionnaires/compose/page.tsx` + `components/admin/questionnaires/compose/{compose-studio,structure-preview}.tsx`): brief → live streaming preview → conversational refine → Open in editor. `<FieldHelp>` on the brief/title fields.

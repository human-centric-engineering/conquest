# F10.4 — Report delivery polish

> Committable tracker for **F10.4**. Three respondent-facing refinements to the completion screen and
> report PDF: don't offer the report download until the report exists, clarify the transcript button,
> and caveat reports generated from a partially-complete questionnaire.

## Intent

- The **Download PDF** button appeared as soon as the completion screen loaded, even while the AI report
  was still generating — offering a "report" download with no report in it.
- The conversation-download button read **Transcript**, ambiguous next to the report download.
- A session can be submitted early (before all slots are answered). A report generated from a thin set
  of answers should say so, so the respondent knows to weight it accordingly.

## What shipped

1. **Download gated on report-ready** — in an AI report mode (`raw_plus_insights` / `narrative`) the
   completion screen holds **Download PDF** until `insights.status === 'ready'`; raw/disabled modes
   download the answers PDF immediately as before (`session-complete.tsx`).
2. **"Chat Transcript" label** — `TranscriptDownload`'s idle label is now "Chat Transcript"
   (`transcript-download.tsx`); shared by the completion screen and the mid-conversation lifecycle bar.
3. **Partial-report caveat (deterministic)** — the pipeline records `completionPct` (answered / total
   slots, frozen at generation) on `AppRespondentReport`. Below `PARTIAL_REPORT_THRESHOLD_PCT` (75)
   both renderers show a caveat subtitle naming the exact % (`partialReportCaveat`, `report/content.ts`).
   Threaded like `formatted`: generation → worker → `view.ts` (`insights.completionPct`) → on-screen
   `ReportInsights`; and via `SessionExportModel.insightsCompletionPct` → PDF `InsightsSection`.

## Key decisions

- **The caveat is deterministic, not agent-generated.** A factual disclaimer with a precise % and fixed
  wording must not drift, so it is computed and rendered, never produced by the writer or formatter. The
  Report Formatter is specifically the wrong place — its fidelity guard _rejects_ added content, so an
  injected caveat would be discarded. (Decision requested by the user; recorded here.)
- **`completionPct` is frozen at generation**, not recomputed at render — a completed session takes no
  more answers, so generation-time completion is the honest figure and cannot drift.
- **`buildSessionExportPdfModel` report params → options object.** Adding `completionPct` would have made
  a 5th trailing positional param (two adjacent booleans already flagged by review as transposition-prone),
  so the report-embed fields are now one `SessionReportEmbed` object — swaps become structurally impossible.

## Storage

`AppRespondentReport.completionPct Int?` (migration `20260703160000_app_respondent_report_completion_pct`,
hand-authored to avoid the phantom pgvector index-drop; applied via `migrate deploy`). Null for legacy
rows → no caveat.

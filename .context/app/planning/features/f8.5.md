# F8.5 — Per-invitation diagnostics

> Committable tracker for **F8.5**. The live respondent surface is deliberately fail-soft — a turn
> that throws was only `logger.error(...)`'d and lost, so "what went wrong for this invitee?" was
> unanswerable. This adds a persisted error trail + a per-invitation telemetry rollup on a new admin
> **Diagnostics** tab.

## Intent

Give admins a queryable, per-invitation view of token use, response time, cost — and crucially an
**error log** — so a stuck or failed conversation can be debugged after the fact. The telemetry was
already captured per turn (`inspectorCalls`); the net-new capability is **persisting errors** from
the turn pipeline, session creation, invitation delivery, and access refusals.

## Decisions (confirmed with user)

- **Cross-invitation Diagnostics tab** (aggregate + per-invitation table + drill-down), not a fold
  into the existing session detail.
- **Broad capture** — turn + lifecycle + delivery (pipeline throws, persist failures, cost-cap/round
  refusals, session-create failures, invitation email failures).
- **Full deep-dive** — the drill-down shows the raw per-call prompt/response inline (reusing the
  persisted `inspectorCalls`, ungated from preview).
- **Capture is always-on** (no flag — a failure must never be missed); the **tab** is gated on
  `liveSessions`.
- **Privacy:** no low-N k-anonymity suppression (a debug tool keyed on the invitee, who the admin
  already knows), but honour `anonymousMode` by withholding email/name while keeping telemetry/errors.

## What shipped

- **Schema** — new `AppQuestionnaireError` model (versionId FK `onDelete: Cascade`; sessionId /
  invitationId plain pointers; scope/stage/severity/code/message/stack/metadata) + three telemetry
  columns on `AppQuestionnaireTurn` (`durationMs`, `promptTokens`, `completionTokens`). Migration
  `20260626052436_app_questionnaire_diagnostics` (`--create-only`, phantom pgvector DDL stripped).
- **Capture seam** — `lib/app/questionnaire/diagnostics/record-error.ts` (`recordQuestionnaireError`,
  best-effort/never-throws, backfills version+invitation from session, low-PII). Wired into the
  messages route (pipeline throw → graceful `error` frame + record; persist/abuse/sensitivity
  catches; top-level 5xx; cost-cap + round-gate refusals), the session-create route, and the
  invitation send route. Token/duration rollup threaded `persistTurn → recordTurn → row`;
  `totalInspectorTokensIn/Out` helpers added.
- **Aggregation** — `lib/app/questionnaire/analytics/diagnostics.ts` (`getVersionDiagnostics`,
  `getInvitationDiagnostics`) + view types in `views.ts`; barrel re-exports. Per-invitation rollup
  incl. synthetic "(no invitation)" group; version avg+p95 via raw `percentile_cont`.
- **API** — `GET …/versions/:vid/diagnostics` + `…/diagnostics/:invitationId` (admin, master-flag,
  version-scoped); endpoint helpers `versionDiagnostics` / `invitationDiagnostics`.
- **UI** — `diagnostics` tab in `workspace-nav.ts`; version page + `DiagnosticsView` (stat tiles +
  per-invitation table, date-window filter); drill-down page + `InvitationDiagnosticsView`
  (lifecycle, telemetry tiles, error log with stack/metadata expanders, per-turn table with a
  read-only `DiagnosticsInspectorCalls` deep-dive).

gated (`withQuestionnairesEnabled`) like the other analytics routes. No new feature-flag row.

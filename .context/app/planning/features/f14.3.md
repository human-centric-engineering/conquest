# F14.3 — Cohort Report auto-narrative + recommendations/actions

> Committable tracker for **F14.3**. The seeded cohort-report agent turns the F14.1 dataset into a
> woven analytical report — thematic analysis, proposed charts, recommendations and actions — stored
> as version-controlled revisions and surfaced (read + generate) on the round detail page. Completes
> the first shippable slice (F14.1→F14.3): an admin can generate a charted cohort report with little
> input.

## Intent

The cohort report's core: an AI that reads the aggregated, segmented results and writes a report — a
thematic analysis that surfaces what's significant, woven prose, charts it judges worth showing, and
concrete recommendations + actions. It must respect privacy (never leak a suppressed figure) and be
re-runnable, with every iteration version-controlled so later editing (F14.5) and publishing (F14.6)
build on it.

## Decisions

- **Direct-agent generation, not a dispatcher capability** — follows the Respondent Report precedent
  (`report/generate.ts`): resolve the seeded agent → build messages → `runStructuredCompletion`. The
  "new capabilities" the brief envisioned are realised as this generation pipeline + the F14.5
  section-refine; no `AiCapability` row is needed for report writing (matches the codebase).
- **One combined structured output** — thematic analysis + narrative + chart proposal +
  recommendations + actions in a single validated call. Simpler/cheaper than a two-phase round-trip
  and enough for v1; a map-reduce split is a future option for very large rounds.
- **AI proposes charts, bounded to the catalog** — the prompt supplies the exact `questionId`s and
  `dimensionKey`s; `validateCohortReportContent` drops any malformed chart or dangling section
  reference, so a generation can never persist a broken revision.
- **Privacy in the prompt** — the digest surfaces suppressed questions/segments as "hidden/too few",
  never their values, and instructs the agent never to reveal them.
- **Append-only revisions** — every generation appends an `AppCohortReportRevision`; the working head
  is `MAX(revisionNumber)`. Sets up restore/publish (F14.6) and editing (F14.5).
- **Synchronous generate (v1)** — a `POST …/generate` runs + persists + returns the refreshed view
  (admin watches a spinner). Streaming "watch it build" is a deferred enhancement; the structure
  (digest → agent → validate) is already streaming-ready.

## What shipped

- **Schema** — `AppCohortReportRevision` (append-only, `@@unique([reportId, revisionNumber])`) +
  `revisions` relation; `COHORT_REPORT_AUTHORS` enum (ai | admin | ai_assist). Migration
  `…_app_cohort_report_revision` (create-only, phantom pgvector DDL stripped, applied via
  `migrate deploy` to avoid the re-diff).
- **Content** — `cohort-report/content.ts`: `CohortReportContent` / `CohortReportSection`,
  `validateCohortReportContent` (coerce/bound, drop bad charts + dangling refs),
  `buildCohortDatasetDigest` + `buildChartCatalogText` (k-anonymity-safe prompt substrate).
- **Generation** — `cohort-report/generate.ts` (`generateCohortReport`): config + dataset digest +
  catalog + optional round/cohort context + client-KB grounding → seeded agent →
  `runStructuredCompletion` → validated content + cost.
- **Persistence + view** — `persist.ts` (`ensureCohortReport`, `appendCohortReportRevision`,
  `markCohortReportFailed`); `view.ts` (`buildCohortReportView` — header + head revision + dataset).
- **Agent** — seed `055-cohort-report-agent.ts` (`app-cohort-report`, internal, restricted KB,
  reasoning-tier, $50/mo cap). Constant `COHORT_REPORT_AGENT_SLUG`.
- **API** — `GET …/cohort-report` (view), `POST …/cohort-report/generate` (flag + per-version-enabled
  gate + per-admin generate sub-cap; audit). Endpoint helpers added.
- **UI** — `CohortReportPanel` (`components/admin/cohorts/`): version picker, Generate/Regenerate,
  markdown narrative + resolved charts (`buildChartData` → `CohortChart`) + recommendations + actions;
  wired into the round detail page behind `isCohortReportEnabled()`.

## Gates

`isCohortReportEnabled()` (master + cohorts + cohort-report sub-flag) gates both routes; the generate
route additionally requires the per-version `config.cohortReport.enabled` toggle and the seeded agent.

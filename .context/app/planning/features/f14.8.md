# F14.8 — Version-wide synthesis + streamed generation

> Committable tracker for **F14.8**. Two additive capabilities on top of the round cohort report:
> (1) generation now **streams** its phases over SSE instead of blocking on one ~90s call, and
> (2) the same AI report can be generated **version-wide** — across every round AND open-ended
> (non-round) sessions — not just per-round.

## Intent

The cohort report (F14.3) already does AI thematic synthesis (summary, narrative sections, charts,
recommendations, actions) over a **round's** submissions. Two gaps remained:

- **Black-box generation.** The generate route was fully synchronous; the admin watched one spinner
  for the whole LLM call. F14.3 flagged streaming as a future enhancement.
- **No cross-round view.** Reports were `roundId @unique` (1:1 with a round). There was no synthesis
  spanning all rounds of a version, or covering open-ended sessions.

Both are cheap because the generator was already ~95% scope-agnostic: round-scoping lived in a single
session `where` clause.

## Decisions

- **Polymorphic owner, not a parallel feature.** Generalise `AppCohortReport` to `scopeKind` =
  `round | version` rather than building a second report model — the dataset, segmentation,
  k-anonymity, content schema, agent, prompt, revisions, publish, PDF and search are all reused. The
  round↔version difference is encoded once in `lib/app/questionnaire/cohort-report/scope.ts`
  (`ReportScope`, `scopeSessionWhere`, `scopeOwnerWhere`, `scopeOwnerCreate`).
- **Nullable-unique owner keys (no partial index).** `roundId` stays `@unique` but becomes nullable
  (Postgres multi-NULL unique → one report per round, round FK/cascade preserved); a new
  `versionOwnerId @unique` (= versionId) gives one report per version. `scopeKind DEFAULT 'round'`
  means **no backfill** — existing rows classify themselves.
- **Full parity for version-wide.** The version report gets the same editor / revisions / publish /
  PDF surface as the round report, via a version route family under
  `app/api/v1/app/questionnaires/[id]/versions/[vid]/cohort-report/**` and a **Report** tab in the
  version workspace. The Analytics tab links across to it.
- **Streaming = phase generator, sync kept.** `streamGenerateCohortReport` (async generator yielding
  `ReportGenProgressEvent`) is the new core; `generateCohortReport` is a thin drain-wrapper so existing
  callers/tests are untouched. `streamReportRun` wraps it for a route (persist on `done`, mark `failed`
  on `error`); the `…/generate/stream` routes return `sseResponse(...)`. The synchronous `…/generate`
  routes remain for back-compat.
- **No verbatim quotes.** Inherits the cohort report's aggregated-themes-only privacy stance (the
  earlier "representative verbatim quotes" idea was dropped — it conflicts with that stance).
- **No new agent/flag/seed.** Reuses the seeded `app-cohort-report` agent, the cohort-report feature
  flag, and the per-version `config.cohortReport.enabled` opt-in.

## Surface

- **Schema:** `AppCohortReport` gains `scopeKind`, `versionOwnerId @unique`, nullable `roundId`.
  Migration `20260626120000_app_cohort_report_polymorphic_owner` (no backfill; pgvector phantom DROPs
  stripped).
- **Lib:** new `scope.ts`, `report-events.ts`, `stream-run.ts`; `generate.ts` (streaming generator),
  `dataset.ts`/`persist.ts`/`view.ts`/`content.ts`/`types.ts` made owner-agnostic.
- **Routes:** round `…/cohort-report/generate/stream`; version family
  `…/versions/[vid]/cohort-report/{route,generate,generate/stream,revisions,publish,export.pdf,dataset}`.
  Cross-report search (`/cohort-reports/search`) now includes version-scope rows.
- **UI:** `cohort-report-panel.tsx` owner-agnostic (`ReportApi`) + SSE phase line; version workspace
  **Report** tab; Analytics-tab teaser link.

## Verification

Type-check clean (full project); 103 cohort-report unit tests + ~2,946-test regression sweep green;
migration applied with no schema drift (pgvector indexes intact); round cohort-report flow unchanged.

# F7.8 — Slot overview minimap + after-turn stepper

> Committable tracker for **F7.8**. For long questionnaires with many data slots the respondent
> couldn't see overall fill state (the right panel scrolls) and could miss a slot a turn filled below
> the fold. This adds an at-a-glance overview map and an after-turn scroll-to-and-step affordance.
> **Data-slot mode only** — question mode is untouched.

## Intent

Give the respondent a one-glance picture of which slots are filled (and how confidently) without
scrolling, and make sure a turn's captures are seen even when they land off-screen — while keeping the
panel's calm "felt, not totted up" voice.

## Decisions (confirmed with the user)

- **A true minimap, not a header block-grid.** A first cut put a sticky themed block-grid (with theme
  labels + a legend) above the list; the user rejected it. What they wanted is a **floating, vertical,
  scaled-down mirror of the scroll area — exactly like the Sunrise workflow-canvas minimap**: one bar
  per slot proportional to the real rows, a **viewport window** rectangle that moves as the list
  scrolls, click/drag to scrub. **No theme headers, no legend** — purely a graphic.
- **Placement: floating overlay** pinned to the left edge of the data-slot scroll area.
- **Bars: confidence-tinted when filled, faint when empty** (conveys completed-vs-uncompleted plus
  the confidence nuance, no legend needed).
- **After-turn behaviour: auto-scroll + stepper footer** — scroll to the topmost newly-filled slot,
  pulse it, and show "N more answers recorded →" that steps through the rest.
- **Scope: data-slot mode only.**

## What shipped

### Minimap (`components/app/questionnaire/panel/slot-minimap.tsx` + `lib/app/questionnaire/panel/minimap.ts`, new)

- `computeMiniMapModel` (pure) projects measured pixel geometry — content height, viewport height +
  offset, and each row's `{top, height}` — into a percentage-based model: one `bar` per slot
  (`topPct`/`heightPct`, `filled`, confidence `band`), plus the `windowTopPct`/`windowHeightPct`
  viewport rectangle, and an `overflow` flag. A `MIN_BAR_PCT` floor keeps single-line rows visible.
- `SlotMiniMap` (presentational) renders the bars absolutely-positioned by those percentages
  (confidence-tinted when filled, faint when not; `newlyFilledKeys` pulse) and the viewport-window
  rectangle. Pointer down/move/up on the track call `onScrubToFraction(fraction, smooth)` to scrub
  the list; `aria-hidden` (purely a visual scroll aid). Floats at the left edge of the scroll area.

### Panel wiring (`components/app/questionnaire/panel/answer-slot-panel.tsx`)

- Data-slot rows extracted to a `DataSlotRow` carrying a stable `id={panelSlotDomId(key)}` +
  `data-slot-key` anchor, `tabIndex={-1}`, a highlight ring, and the stepper footer.
- `measureGeometry` reads the scroll container + every `[data-slot-key]` row's rect into content-pixel
  space; runs on content change (`view`) and a `ResizeObserver`. A scroll only updates the cheap
  `viewportTop`. `computeMiniMapModel` turns these into the minimap model; the minimap renders in a
  `relative` wrapper around the scroll container, only when slot count `> OVERVIEW_MIN_SLOTS` (10)
  **and** the content overflows. `scrubToFraction` scrolls the container to the clicked fraction.
- `scrollToSlot(key)` scrolls the panel's **own** container (`scrollTo` off `getBoundingClientRect`
  deltas — never the window), focuses the row, pulses it (~1.5s), and emits an `aria-live`
  announcement. No-ops when the panel isn't laid out (hidden below `lg`).
- Stepper state machine: a new `newlyFilledKeys` set arms the cursor at 0 and scrolls to the first;
  the footer on the focused row shows "N more answers recorded" (singular "1 more slot was answered"
  on the last hop); clicking advances + scrolls. A new turn's keys restart it.
- Honors reduced motion via `usePrefersReducedMotion` (new hook, `lib/hooks/`) for the JS scroll, and
  `motion-reduce:` variants for the CSS pulses.

### Newly-filled detection (`lib/app/questionnaire/panel/newly-filled.ts`, new)

- Pure helpers: `panelSlotDomId`, `dataSlotKeysInOrder`, and `diffNewlyFilled(prev, next)` — diffs two
  panel snapshots in display order; a slot is "newly filled" when it went unfilled→filled **or** its
  `answeredAtTurnIndex` advanced (catches refinements, value changes, provisional→confident). `prev`
  null (first/SSR seed) returns `[]` — never auto-scrolls the seed.
- `SessionWorkspace` keeps a `prevPanelRef`, runs the diff on each `panel.view` change (data-slot mode
  only), and feeds the ordered keys into `AnswerSlotPanel`. The stream never exposes a turn ordinal,
  so a snapshot diff is the reliable client signal.

### Data plumbing

- `DataSlotPanelSlot.answeredAtTurnIndex` added (`panel/types.ts`); resolved in the read seam
  (`_lib/answer-panel.ts`) from the fill's `lastUpdatedTurnId` via the existing turn-ordinal map,
  mirroring question slots. **No migration** — the column already existed.

## Tests

- **Unit:** `newly-filled` (display order, seed-returns-empty, unfilled→filled, no-change,
  refinement/turn-advance, provisional→confident, null/question-mode guards); `minimap`
  (`computeMiniMapModel`: overflow detection, percentage projection, band assignment, min-bar floor,
  zero-height/clamp safety); `SlotMiniMap` (bar count + position, band tints vs faint, viewport
  window, newly-filled pulse, click-scrub + drag-scrub); `AnswerSlotPanel` (minimap hidden without
  overflow, renders + scrubs under layout stubs, stepper footer copy arms + decrements + clears,
  `usePrefersReducedMotion`).
- **Component:** `SessionWorkspace` — first snapshot seeds silently; the next snapshot's fill
  propagates `newlyFilledKeys` to the panel.
- **Integration:** `answer-panel.test.ts` — data-slot `answeredAtTurnIndex` resolves from
  `lastUpdatedTurnId` (and stays null when absent/unmapped).
- `npm run validate` green; affected questionnaire suites pass.

## Not in scope

- A mobile/`< lg` surface — the panel is `hidden lg:flex`, so the aids no-op there (effects bail on
  zero layout height). The full-width chat remains the small-screen experience.
- Question-mode (section) navigation — deferred; the "many slots" pressure is in data-slot mode.

# F7.4 — PDF export

> Committable tracker for **F7.4**. A completed session's answers download as a branded
> PDF — the takeaway artefact that closes the demo. Two surfaces: a respondent button on
> the F7.3 completion screen, and an admin API route ready for a P8 admin session view.
> App dependency `@react-pdf/renderer` (vertical — not promoted to Sunrise). No migration
> (reads existing session/answer rows).

## Intent

F7.1 rendered the conversation, F7.2 the captured answers, F7.3 the lifecycle through to
`completed`. F7.4 lets the respondent (and, later, an admin) **take the result away** as a
readable, branded PDF — satisfying P7's definition of done ("PDF export of results works").

## Decisions (confirmed with the user)

- **Both surfaces now.** Respondent download on the completion page (makes the demo
  end-to-end green today) **and** an admin API route. The admin route has no UI trigger
  yet — P8's admin session views will wire a button to it; building the backend now keeps
  that work pure front-end.
- **Detailed content.** Per question: prompt, captured answer, a quiet provenance ·
  confidence line, the agent's rationale, and the refinement-history audit trail.
- **Full coverage.** Every question is rendered grouped by section; unanswered slots show
  "Not answered". The export builds its own view with `full_progress` scope, so the
  version's `answerSlotPanelScope` (which may be `answered_only` for the live panel) never
  shortens the record.
- **Anonymous redaction at the boundary.** When the version is `anonymousMode`, the model
  builder drops respondent identity (header reads "Anonymous respondent") and the seam
  never even queries the name. Honoured now, not deferred to F8.3.
- **Name only, never email.** A non-anonymous export shows the respondent's display name;
  email is not included.

## What shipped

### Pure core (`lib/app/questionnaire/export/`, Prisma-free, Next-free)

- `types.ts` — `SessionExportModel` (header metadata + resolved theme + `PanelSectionView[]`).
- `build-session-export-model.ts` — `buildSessionExportModel`: reuses `buildAnswerPanelView`
  (forced `full_progress`) for the section/slot join and `resolveTheme` for branding;
  applies anonymous redaction and a one-line audience summary. Clock-free (timestamps in).

### Document + render (server-only)

- `components/app/questionnaire/export/session-pdf-document.tsx` — the `@react-pdf/renderer`
  `<Document>` (NOT a client component): branded header (logo, accent rule, title, version,
  goal, audience, respondent, completion date, progress), section headings, per-slot answer +
  meta + rationale + refinement history, fixed footer with generation date + page numbers.
  Reuses the F7.2 `formatAnswerValue`.
- `_lib/render-session-pdf.tsx` — `renderSessionPdf(model)` → `renderToBuffer(...)`.

### DB seam (`questionnaire-sessions/_lib/`)

- `session-export.ts` — `loadSessionExport(sessionId)` (one query mirroring the answer-panel
  loader, extended with title / version / goal / audience / `anonymousMode` / demo-client
  theme / latest `completed` event; respondent name looked up only when not anonymous) +
  `buildSessionExportPdfModel(loaded)` (best-effort logo fetch → data URI with a 3s timeout
  and 1 MB cap, stamps `generatedAt`, delegates to the pure builder). Runs the logo fetch
  only **after** the route authorises.
- `pdf-response.ts` — `sessionPdfResponse(buffer, model)`: `application/pdf`, `attachment`
  filename `responses-<slug>-v<N>.pdf`, `Cache-Control: no-store`.

### Routes (dot-segment, `runtime = 'nodejs'`)

- **Respondent:** `GET …/questionnaire-sessions/:id/export.pdf` — `withLiveSessionsEnabled` +
  `resolveTurnAccess` (authed owner OR anonymous `X-Session-Token`). Gate order
  flag→load→access→build→render.
- **Admin:** `GET …/questionnaires/:id/sessions/:sessionId/export.pdf` —
  `withQuestionnairesEnabled` + `withAdminAuth`; ownership check (session's version must
  belong to the URL questionnaire, else 404, no render).
- Registry: `exportPdf` on `QUESTIONNAIRE_SESSIONS`, `sessionExportPdf` on `QUESTIONNAIRES`.

### Respondent UI

- `SessionComplete` (F7.3) gains a "Download PDF" button — `fetch` with the `X-Session-Token`
  header (a plain `<a download>` can't set headers for the no-login path), then a blob
  object-URL download; a transient error line on failure. `SessionWorkspace` threads
  `sessionId` + `accessToken` in.

## Tests

- **Unit (pure):** `export/build-session-export-model.test.ts` — full coverage (answered +
  unanswered), anonymous redaction, theme resolution, audience summary, header passthrough.
- **Unit (render):** `questionnaire-sessions/render-session-pdf.test.tsx` — real
  `renderToBuffer` produces a `%PDF` byte stream for the answered + anonymous variants.
- **Integration (routes):** respondent `export-pdf-route` (gate order, both access modes,
  no build/render before access, PDF envelope + filename); admin `session-export-pdf-route`
  (gate order, admin auth, ownership 404, anonymous-model passthrough).
- **Component:** `session-complete` extended — fetch + blob download, `X-Session-Token` for
  anonymous (absent for authed), transient error.

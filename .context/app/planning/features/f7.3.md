# F7.3 ‚Äî Session lifecycle UX

> Committable tracker for **F7.3**. The respondent-facing window onto the session
> lifecycle: pause/resume, the completion-offer ‚Üí submit flow, the cost-cap state, and the
> anonymous-mode indicator. Closes the one missing path ‚Äî a live respondent session had no
> way to reach `completed`. Gated by `APP_QUESTIONNAIRES_LIVE_SESSIONS_ENABLED`; no
> migration (every status + event already exists from F4.6).

## Intent

F7.1 rendered the conversation; F7.2 made the captured answers visible. F7.3 makes the
session's **lifecycle** visible and actionable: the respondent can pause and resume, sees
when the agent offers to submit and can accept it, sees a calm confirmation when done, and
(for anonymous runs) a quiet "responses are anonymous" indicator.

## The gap it closed

Before F7.3 the orchestrator streamed a completion _offer_ as prose but nothing recorded the
acceptance ‚Äî the only `completed` transition was the admin-only `/complete` preview route.
F7.3 adds the respondent **submission flow** as the sole live path to `completed`.

## Decisions (confirmed with the user)

- **Offer signal = a status read endpoint, not a stream frame.** The F6.1 SSE stream carries
  no marker distinguishing an offer turn from a question turn. Rather than reopen that
  contract (the F7.2 decision), a new `GET ‚Ä¶/status` is refetched on the same `onTurnSettled`
  that drives the answer panel; the Submit affordance appears when `completion.kind === 'offer'`.
- **Submit gate is upstream.** `assessCompletion` only returns `offer` once the required
  gate is clear, so the Submit affordance never appears with a required question blank ‚Äî the
  submit endpoint just re-asserts the offer state via `resolveCompletion` (**no contradiction
  sweep**; contradictions surface live during chat). The lone exception is the existing F4.5
  "a question-cap-reached session can always submit" behaviour, honoured as-is.
- **Pause is signed-in only.** A no-login session's token is client-only (lost on reload), so
  a deliberate pause has nowhere durable to resume from ‚Üí the endpoint refuses anonymous
  callers with `403 PAUSE_NOT_PERMITTED`. They still see system-driven states.

## What shipped

### PR1 ‚Äî backend (read + lifecycle + submit)

- **Pure** `lib/app/questionnaire/session/status-view.ts`: `buildSessionStatusView` (maps the
  F4.5 assessment + F6.3 tier + status into a respondent-safe `SessionStatusView` ‚Äî coarse
  cost **tier** only, never the spend) + `canSubmitSession`. Exported via `session/index.ts`.
- **Seam** `_lib/session-status.ts`: `loadSessionStatus` reuses `buildTurnContext` (assessment
  byte-identical to the live turn), `sumSessionTurnCost`, `classifyCostCap`; cost reported only
  when a positive budget is set **and** enforcement is enabled.
- **Routes** under `questionnaire-sessions/[id]/`: `GET status` (both kinds), `POST lifecycle`
  `{ action: pause|resume }` (signed-in only; `resume` returns resume state), `POST submit`
  (both kinds; idempotent on completed; `409 SUBMIT_NOT_READY` / `SESSION_NOT_ACTIVE`). All
  reuse `resolveTurnAccess`; gate order flag‚Üíload‚Üíaccess‚Üíaction; an illegal transition ‚Üí 409.
- **Registry** entries `status`/`lifecycle`/`submit` on `API.APP.QUESTIONNAIRE_SESSIONS`.

### PR2 ‚Äî frontend (lifecycle UI + confirmation)

- **Stream hook**: added `completed` to `QuestionnaireChatStatus` + `BLOCKING_STATUSES`, and an
  imperative `applyStatus(status)` so lifecycle actions push authoritative status into the
  shared stream (composer enables/disables in lockstep).
- **`useSessionLifecycle`** hook (sibling to `useAnswerPanel`): holds the status view (refetched
  on settle), exposes `pause`/`resume`/`submit` + derived `canSubmit`/`canPause`/`canResume`,
  pushes status via `applyStatus`, surfaces friendly action errors.
- **`SessionWorkspace`** lifts the third hook; the shared `onTurnSettled` now refetches both the
  panel and the status view; swaps to `SessionComplete` on `completed`.
- **Components** (`components/app/questionnaire/lifecycle/`): `SessionLifecycleBar` (anon badge,
  pause/resume, soft-cost hint, action error; renders nothing when empty), `CompletionOffer`
  (Submit CTA + dismiss), `SessionComplete` (themed confirmation).
- **Pages**: authed page SSR-seeds the status view (`loadSessionStatus`) and maps it to the
  initial chat status (budget-pause ‚Üí `cost_capped`, respondent pause ‚Üí resumable
  `not_active`, completed ‚Üí confirmation); the anonymous boot fetches on mount.

## Tests

- **Unit (pure):** `session/status-view.test.ts` (projection, cost-null-when-uncapped,
  tier-only, `canSubmitSession` matrix incl. the capped edge).
- **Integration (routes):** `status-route` (gate order, both access modes, tier-not-spend),
  `lifecycle-route` (gate order, anon-pause 403, pause/resume, illegal‚Üí409, validation),
  `submit-route` (gate order, anon allowed, idempotent-completed, paused 409, not-ready 409,
  happy path, transition race).
- **Hook:** `use-session-lifecycle` (SSR seed, anon fetch + header, derived flags incl.
  budget-pause non-resumable, pause/resume/submit push status, error surfaced no push);
  `use-questionnaire-session-stream` extended (`applyStatus` terminal + clear-on-resume).
- **Component:** `session-lifecycle-bar` (empty render, anon, pause/resume, cost hint, busy,
  error), `completion-offer` (submit, busy, dismiss), `session-complete` (count copy).

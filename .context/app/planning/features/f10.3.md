# F10.3 — Report Formatter (second-pass formatting agent)

> Committable tracker for **F10.3**. Splits report generation into two agents: the existing WRITER
> (grounded substance) and a new FORMATTER (form only). Replaces the blunt render-time
> `splitReportParagraphs` sentence-regrouping — which chops a wall of text into too many uniform
> paragraphs — with a semantic pass that re-paragraphs naturally, converts inline dash-runs to bullets,
> and strips AI-isms (em-dash overuse, flowery filler). Ships behind a flag, off by default.

## Intent

The writer agent usually returns a wall of text; the deterministic `splitReportParagraphs` then
re-groups it into uniform ≤3-sentence / ≤280-char chunks with no regard for where a paragraph
_semantically_ ends — the "too many paragraphs" symptom. Paragraph judgement, de-slopping, and
dash→bullet conversion are semantic tasks a regex can't do and that the content agent self-polices
poorly mid-generation. F10.3 adds a dedicated single-remit formatter that runs on the writer's output.

## Agreed scope

- **New formatter agent** `app-report-formatter` (`REPORT_FORMATTER_AGENT_SLUG`), seeded by
  `061-report-formatter-agent.ts`, `visibility: 'internal'`, `temperature: 0.2`, empty provider/model
  resolved at the cheaper **`chat`** tier (formatting is mechanical; reports are async so latency is free).
- **Generic utility** `formatReportContent(content, { format })` (`lib/app/questionnaire/report/format.ts`)
  over the shared `summary / sections[{heading,body}] / actions` core, with a `plaintext | markdown`
  flag so the Cohort Report can adopt it later. Runs the shared `runStructuredCompletion` direct-agent
  pattern. **Never throws.**
- **Fidelity guard (load-bearing)** — verifies the output preserved structure (same section count,
  headings in order, action count; non-empty blocks via `validateRespondentReportContent`). On any
  drift, parse failure, timeout, provider error, or missing agent → returns the **original content
  unchanged** with `formatted: false`. `actions` pass through verbatim.
- **Flag** `APP_REPORT_FORMATTER_ENABLED` (`062-report-formatter-flag.ts`), **off by default** —
  ship-dark so the two-agent output can be compared before rollout.

## What shipped (by phase, one commit)

1. **Formatter agent + flag** — `REPORT_FORMATTER_AGENT_SLUG` + `APP_REPORT_FORMATTER_FLAG` constants;
   seeds `061` (agent) and `062` (flag, disabled).
2. **Formatter utility** — `formatReportContent` + `preservesStructure` fidelity guard.
3. **Storage** — `AppRespondentReport.formatted Boolean @default(false)` (migration
   `20260703120000_app_respondent_report_formatted`, hand-authored to avoid the phantom pgvector
   index-drop; applied via `migrate deploy`).
4. **Generation wiring** — `generateRespondentReport` resolves the flag once; when on it runs the
   formatter after validation, sums both costs, and returns `formatted`. Agent 1's prompt is **thinned**
   when the flag is on (`PARAGRAPH_RULES_LIGHT`; `structured` style drops the manual bullet mechanic).
   The worker persists `formatted`.
5. **Trust-mode render** — `splitReportParagraphs(text, { trustParagraphs })` runs pass 1 only (honour
   authored blank-line breaks + bullet blocks, skip sentence re-grouping). `formatted` threaded through
   `view.ts` → on-screen `SessionComplete`, and through `SessionExportModel.insightsFormatted` →
   `buildSessionExportPdfModel` → PDF `InsightsSection`/`Paragraphs`. `formatted: false` (flag off,
   fallback, legacy rows) keeps the deterministic split — the splitter is demoted to fallback, never removed.

## Key decisions

- **Separation of concerns** — writer owns grounded substance, formatter owns form. Each prompt is
  simpler; the writer sheds layout responsibility only when the formatter is enabled.
- **Fidelity over polish** — a formatter that drifts is discarded, not stored. The deterministic split
  remains the safe fallback and still fixes legacy/un-formatted rows at render time.
- **Generic, cohort-ready** — the utility is report-kind-agnostic; the Cohort Report seam (opt-in
  "Polish formatting" editor action, pre-`markdownToHtml`) is documented but **not** wired in this work.
- **Chat tier** — the formatter uses the cheaper default model; async generation makes the extra latency free.

## Tests

`format.test.ts` — happy path preserves structure + `formatted: true`; actions pass through verbatim;
fallback on section-count / heading / action-count drift, on unparseable output (both attempts), on
provider error, and on missing agent; resolves at the `chat` tier; plaintext vs markdown convention.
`generate.test.ts` — flag off ⇒ no formatter call + `formatted: false`; flag on ⇒ formatter called
with the writer output, costs summed, fallback propagated, prompt thinned (light paragraph rule present,
strict rule absent), structured bullet mechanic dropped. `content.test.ts` — `trustParagraphs` honours
authored breaks without re-chopping, doesn't sub-split a wall, preserves bullet blocks, normalises CRLF.
`npm run validate` + report/export suites clean.
